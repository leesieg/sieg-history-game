# 高保真版本完整机制接入设计

## 目标

将 `prototype/demos/帝国的代价-微信小游戏demo2.html` 的完整游戏机制迁移到高保真界面，使高保真版本成为后续唯一维护版本。

## 已确认边界

| 项目 | 决定 |
|---|---|
| 主界面 | 保留现有高保真 HUD 布局与视觉 |
| 扩展界面 | 允许新增外交、战争、事件、选举等专用弹窗和抽屉 |
| Demo2 | 永久冻结，只作为迁移来源和行为基线 |
| 后续维护 | 机制只维护高保真版本 |
| 文件结构 | 拆分 HTML、CSS、数据、机制和 UI 适配代码 |
| Git | 每个阶段独立提交，可单独回滚 |

## 核心架构

```text
玩家操作
  ↓
高保真 UI 适配层
  ↓
机制命令
  ↓
唯一世界状态
  ↓
季度结算与历史因果
  ↓
HUD / 地图 / 抽屉 / 弹窗重新渲染
```

机制层不得直接访问 DOM；界面层不得直接修改世界状态。所有状态变化通过命令函数进入机制层。

## 迁移原则

1. 不嵌入 Demo2 页面，不使用 iframe。
2. 不保留两套世界状态。
3. 不把 Demo2 整页脚本直接挂到高保真 DOM。
4. 先用行为测试固定机制，再逐模块迁移。
5. 旧高保真 HTML 保留为静态快照。
6. 新版本每个阶段都必须可运行、可测试、可回滚。

## 验收标准

- Demo2 SHA-1 始终为 `7a3fc0692ae10cca4dad440e6ffd82d7031ddfed`。
- 全仓测试与高保真工业时代长回合测试全部通过。
- 新高保真版本覆盖 Demo2 全部可操作机制。
- 地图拖动、缩放、选格、军团选择互不冲突。
- 所有按钮接入真实机制，不保留演示 Toast。
- 桌面与横屏手机无关键组件重叠。
- 每阶段对应一个独立 Git 提交。
