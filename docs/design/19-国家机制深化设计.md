# 19 · 国家机制深化设计（流模型重述）

> 适用范围：`prototype/hifi/` 主线的「国家」面板（政体 / 统治者 / 王权·合法性 / 政制 / 议会 / 决议）。
> 前置盘点见 [`18-hifi交互组件全盘点.md`](18-hifi交互组件全盘点.md)；本文是机制深化的设计依据，实现随后分批接入引擎。

## 0. 一句话目标

把「法律能不停点但没有任何影响、阶层权力只是摆设、王权除了一个决议门槛毫无后果」这些**空机制**，重写成符合项目核心模型的、互相牵制的系统。

## 1. 核心模型：基底 → 流 → 影响/事件

项目的统一世界观（与 demo2 三层因果引擎一脉相承）：

```
地块 / 人口 / 资源（基底）
        │  产出、流动、扩散
        ▼
   各种「流」（贸易流 / 产出流 / 人口流 / 科技流 / 信仰流 …）
        │  累积、传导
        ▼
   压力层（pressures）→ 转折层（eras / situations / events）
```

**一切机制（政治 / 外交 / 军事 / 经济）的本质，是流的调节阀：要么改变某条流的大小/方向/效率，要么被某条流反向驱动。** 国家面板里的政治机制也不例外——它们是「分配产出流、汲取产出流、决定谁满意」的杠杆。

### 1.1 hifi 现有的「流」清单（复用既有词汇，勿自造）

| 流 | 现有载体 | 基底来源 |
|---|---|---|
| 产出流（粮/钱/军需） | `economy.tileOutput` → `settleCountry` | 地块 population × good × control × (1-devastation) |
| 贸易流 | `trade.route.flow` / `route.boost` | 地块节点 + 路线 value/cost |
| 科技扩散流 | `history.spreadTechnology` → `technologyAwareness%` | 邻接 + 压力 |
| 探索流 | `pressures.exploration` → `exploration.points` | 沿海/纪元 |
| 信仰流 | 地块 `religion` + 宗教法律（待接） | 地块宗教分布 |
| 人口流 → 军队 | `warfare.mobilizeArmy`（消耗地块 POP） | 地块 population |

压力层：`country.pressures = {trade, military, fiscal, exploration, faith, ideas}`。
转折层：`history.eras`（feudal→discovery→faith→absolutism→revolution→industrial）、`situations`（黑死病/小冰期）。

## 2. 国家机制如何接入流模型

### 2.1 王权 centralPower（0–100）

**定位**：决定「中央能从产出流里直接汲取多少」对「被阶层截留多少」的分配阀。

- **被什么流影响**：阶层权力总和上升 → 王权自然回落（集权与阶层此消彼长）；合法性低 → 王权增长受阻。
- **影响什么流**：
  - 高王权（≥70）：中央直辖产出流 +%（`settleCountry` 中央留存比例↑）、行政点产出 +；**代价**：阶层满意每季自然 −（集权压迫）。
  - 低王权（≤30）：阶层权力增长加速、部分集权决议锁死、统一税制等法律不可用。
- **展示**：tooltip 给出「当前分配比例、下季预计变化、阈值后果」。

### 2.2 合法性 legitimacy（0–100）

**定位**：压力层的总闸——低合法性让所有流普遍打折并触发事件。

- **被什么流影响**：议会通过 +3（现有）、史诗使命奖励（现有）、黑死病等事件增减（现有）、阶层满意均值低 → 每季自然下滑（新）。
- **影响什么流**：`<55` 时产出流 ×0.95 并提高负面事件概率（现有仅预警，需接乘数）；高合法性缓冲事件。
- **展示**：tooltip 列「当前影响因素清单 + 下季预计 ± + <55 的后果」。**纯聚合，可立即做（效果已大体存在）。**

### 2.3 法律 laws —— 给每条真实 trade-off（调节具体流）

法律是「定下来很久不动、但持续改写某条流系数 + 某阶层满意」的慢变量。当前 `setLaw` 只改字符串、无下游——本节定义下游。

| 类别 | 选项 | 调节的流 | 取舍（阶层/前置） |
|---|---|---|---|
| **税收** taxation | 传统 customary | 金钱产出流 ×1.00 | 基准 |
| | 阶层免税 estate_exemptions | 金钱产出流 ×0.85 | 贵族 + 教会满意 +8 |
| | 统一税制 uniform | 金钱产出流 ×1.15 | 贵族满意 −10、平民 +4；**需王权≥50** |
| **动员** mobilization | 有限 limited | 人口流→军队 基准 | 基准 |
| | 征召 levy | 募兵 POP 成本 ↓、上限 ↑ | 平民满意 −6 |
| | 常备 standing | 解锁专业兵质量 +、军需维护流 + | **需财政改革≥2** |
| **宗教** religion | 宽容 toleration | 异端地块稳定（产出流不打折） | 教会/教士满意 −6 |
| | 正统 orthodoxy | 同信仰地块满意 + | 异端地块不满 +、信仰流收紧 |
| | 改革宗 reformed | 切换时信仰流重置 | **需 reformation flag**；切换冲击合法性 − |
| **权力** authority | 王朝 dynastic | 王权自然维持 | 基准 |
| | 公民 civic | 商人/市民满意 + | 王权上限封顶 |
| | 宪政 constitutional | 合法性 +、议会支持 + | 王权封顶 ~60 |
| | 绝对 absolute | 王权 + | 贵族满意 −；**需财政改革≥3** |

实现：法律值进 `tileOutput`/`settleCountry`/`mobilizeArmy` 的系数表（如 `LAW_EFFECTS`），切换时即时结算阶层满意增减。

### 2.4 改革槽 reforms（0–5）—— 给每级解锁真实加成

当前除 `political`（×5 喂议会、/2 喂外交容量）外几乎只是决议门槛，`administrative/military/maritime` 纯花钱无产出。改为**每级线性加成**：

| 改革 | 每级效果（×等级） | 满级额外 |
|---|---|---|
| 行政 administrative | 地块控制度上限 +、整合效率 + | 解锁 +1 行政点/季 |
| 财政 fiscal | 金钱产出流 +3%/级 | 解锁财政路线决议 |
| 军事 military | 军需产出流 +/单位质量 + | 解锁常备军路线 |
| 政治 political | 议会支持 +5/级（保留）、外交容量 +（保留） | 解锁立宪/共和决议 |
| 宗教 religious | 信仰流稳定 +、异端不满 − | 解锁宗教决议 |
| 海事 maritime | 贸易流/探索流 + | 解锁海外特许 |

### 2.5 阶层 estates —— 被流喂养、反向惩罚流

每个阶层有 `power(0–100)` 与 `satisfaction(−100..100)`。

- **被什么流影响**（满意度的输入）：产出流分配是否照顾该阶层（免税→贵族/教会喜、统一税→贵族怒）、法律切换、议会让步（让渡特权 → power +2，现有）。
- **影响什么流**（满意度的输出）：
  - 满意度 `< −40`：该阶层关联流每季惩罚——贵族→军需流，商人→金钱流，教会→合法性，平民→粮食流 + **叛乱压力累积**。
  - 满意度高：对应流小幅 +。
- **权力 power 的作用**：决定该阶层在「法律诉求 / 议会表决」里的权重；权力总和与王权此消彼长（见 2.1）。
- **展示**：议会 tab 用**无依赖 SVG 饼图**画权力分布；每个阶层行 tooltip 说明「现在满意/不满会怎样」。

## 3. 核心耦合环（EU 式联动）

```
        统一税制/绝对主义            集权代价
   王权 ───────────────► 金钱产出流↑      王权高 ──► 阶层满意↓
    ▲                                            │
    │ 此消彼长                                    ▼
 阶层权力 ◄──── 让渡特权 ──── 议会支持 ◄── 阶层满意均值 ──► 叛乱压力/合法性
    │                                            ▲
    └────────── 法律诉求权重 ──────────────────────┘
```

读法：想要钱（统一税）就得压贵族满意 → 贵族不满拉低议会支持/堆叛乱压力 → 要么让渡特权（抬阶层权力、削王权）安抚、要么靠高合法性硬扛。每个选择都有代价，没有免费的数字。

## 4. 百科模块 codex（统一概念来源）

新增 `prototype/hifi/scripts/data/codex.js` → `window.HIFI_CODEX`，每个概念一条：

```js
政体: {
  term: "政体",
  summary: "决定权力结构、可用决议路径与阶层构成",
  affectedBy: ["国家决议（可变更政体）"],   // 被什么影响/被什么流驱动
  affects: ["议会是否解锁", "阶层构成", "继承方式"], // 影响什么流/什么机制
}
```

**它是 #1 tooltip、#3 王权/合法性说明、#4 法律/改革效果、#5 阶层作用、#6 决议/整合效果的唯一文案来源**，不再散落在各 render 函数里。

- 交互：`<span class="codex-term" data-codex="政体">封建君主制</span>`，下划虚线；hover/点击弹 popover，内容从 `HIFI_CODEX` 取（`summary` + 受什么影响 + 影响什么）。
- 绑定：事件委托，抽屉每次重渲染后自动生效（无需逐次 rebind）。
