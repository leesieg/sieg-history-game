# 国家政府政治阶层 Demo Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 在现有 HTML demo 中加入完整的国家、政体、改革、议会和地图阶层绑定原型。

**Architecture:** 保持当前单文件静态 HTML demo，不引入框架。新增数据模型、渲染函数和交互面板都放在 `prototype/demos/帝国的代价-微信小游戏demo.html` 内，优先复用现有左上抽屉、右上 HUD、左下地块详情和地图模式组件。

**Tech Stack:** HTML、CSS、原生 JavaScript、SVG 六边形地图。

## 实现边界

这是完整机制的 demo 化实现，不是最终平衡版本。

| 做 | 不做 |
|---|---|
| 支持全部政体类型 | 不做 AI 完整政治决策 |
| 支持核心权力、改革槽、法律、议会、阶层地图绑定 | 不做复杂历史事件数据库 |
| 支持地块主导阶层和阶层控制度展示 | 不重做地图生成算法 |
| 支持政体变更决议 | 不做存档系统 |
| 支持基本危机触发 | 不做完整战役平衡 |

## Task 3: 重做阶层数据

**Files:**
- Modify: `prototype/demos/帝国的代价-微信小游戏demo.html`

**目标：** 把当前 `state.estates = { nobles, merchants, church, peasants }` 改为完整对象数组。

新结构：

```js
state.estates = {
  nobles: { label: "贵族", power: 42, satisfaction: 1, privileges: ["地方司法权"] },
  merchants: { label: "商人", power: 31, satisfaction: 0, privileges: ["行会特许"] },
  church: { label: "教会", power: 34, satisfaction: 1, privileges: ["什一税"] },
  peasants: { label: "平民", power: 28, satisfaction: 0, privileges: [] }
};
```

同步更新旧逻辑：

| 旧逻辑 | 新逻辑 |
|---|---|
| `state.estates.nobles += 1` | `adjustEstate("nobles", { satisfaction: 1 })` |
| `v <= 0` 判断崩溃 | `satisfaction <= -5 && power >= 40` 触发叛乱 |
| 阶层展示单条数值 | 展示权力、满意度、特权 |

验收：

| 检查 | 预期 |
|---|---|
| 阶层抽屉能显示 4 个阶层 | 每个有权力和满意度 |
| 旧危机能正确影响阶层 | 继承危机影响贵族满意度 |

## Task 7: 实现改革和法律操作

**Files:**
- Modify: `prototype/demos/帝国的代价-微信小游戏demo.html`

**目标：** 增加基本改革按钮和法律切换。

行动规则：

| 操作 | 消耗 | 效果 | 代价 |
|---|---|---|---|
| 行政改革 | 1 行动点 + 金钱 | 控制力、整合增强 | 贵族或地方阶层不满 |
| 财政改革 | 1 行动点 | 金钱、税收增强 | 商人权力上升 |
| 军事改革 | 1 行动点 + 军需 | 征兵、军队增强 | 平民满意下降 |
| 宗教改革 | 1 行动点 | 合法性或宽容增强 | 教会权力变化 |
| 政治改革 | 1 行动点 + 合法性 | 解锁路线 | 旧阶层反弹 |
| 海洋改革 | 1 行动点 + 金钱 | 港口、海军增强 | 内陆阶层不满 |

验收：

| 检查 | 预期 |
|---|---|
| 改革会推进槽位 | 数字增加 |
| 改革会影响阶层 | 至少一个阶层满意度或权力变化 |
| 改革不足时决议不可用 | 按钮禁用或提示条件 |

## Task 10: 更新回合结算和危机

**Files:**
- Modify: `prototype/demos/帝国的代价-微信小游戏demo.html`

**目标：** 让政治系统进入每回合结算。

结算规则：

| 结算项 | 逻辑 |
|---|---|
| 阶层权力 | 来自 POP、建筑、地产、地块控制 |
| 满意度漂移 | 高税、征兵、改革、低粮食会影响 |

| 合法性 | 受危机、政体、宗教、首都控制影响 |
| 危机 | 满意度低且权力高的阶层触发 |

验收：

| 检查 | 预期 |

| 阶层叛乱条件可触发 | 日志说明原因 |
| 合法性仍能导致失败 | 保持原结束逻辑 |

## 风险控制

| 风险 | 控制方式 |
|---|---|
| 复杂度太高 | 所有政体共用底层字段，只替换配置 |
| UI 太拥挤 | 左上抽屉分面板，不在地图上堆信息 |
| 阶层和地图脱节 | 每个地块必须有主导阶层和控制度 |
| 政体变更混乱 | 决议显示条件，不满足时不可执行 |
| 老功能损坏 | 每个任务后验证地图点击、缩放、回合结算 |

## 完成标准

| 问题 | Demo 要回答 |
|---|---|
| 我的国家是什么政体？ | 国家面板展示政体、核心权力、法律和改革 |
| 谁在分享我的权力？ | 阶层面板和阶层地图模式展示权力来源 |
| 我怎么改变国家？ | 改革、议会和决议提供可操作路线 |
