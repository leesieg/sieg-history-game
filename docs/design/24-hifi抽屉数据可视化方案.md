# 24 - hifi 抽屉数据可视化方案

> 背景：五个系统抽屉（国家/经济/外交/军事/发展）除「阶层权力」用扇形外，其余模块全是统一的 `label/value` 列表，数据形态丢失。本方案把每个模块换成更贴合数据形态的展示。
>
> 实现：纯函数返回 HTML/SVG 字符串的可复用基元 `prototype/hifi/scripts/ui/widgets.js`（`window.HIFI_WIDGETS`），与已有 `estate-pie` 同思路，无第三方库。CSS 在 `components.css`。

## 一、可复用基元

| 基元 | 函数 | 适用 |
|---|---|---|
| 量表条 | `meter(value, max, {tone, threshold})` | 0–max 有界值，可标阈值 |
| 双向条 | `diverging(value, max)` | ±max（满意度类） |
| 段位点阵 | `pips(value, max)` | 0–N 等级 |
| 雷达图 | `radar([{label,value,max}])` | 多维同量纲 |
| 门槛清单 | `checklist([{label,met}])` | 多条件解锁 |
| 态度色点 | `attitudeDot(attitude)` | 亲→敌六色阶 |
| 占比扇形 | `estatePie`（已有） | 部分占整体 |

## 二、各系统模块映射（已确认「全做」）

### 国家
- 统治者 行政/外交/军事 → **雷达图(3轴)**
- 合法性 → **量表**；改革槽 ×6 → **点阵**；法律 → 分段开关
- 议会支持度 → **量表 + 50 阈值**；阶层满意度 → **双向条(每阶层)**；国内不满 → **量表(红)**
- 国家决议/整合 → 保留按钮（成本/置灰已做）

### 经济
- 贸易政策/关税 → **分段开关**；物价指数 → 基准 1.0 偏离条
- 贸易路线 ×10 → **流量条(meter，按 flow 缩放、盈亏分色)**
- 结构压力 6维 → **雷达图(6轴)**
- 敕令/议程/建设 → 保留按钮

### 外交
- 外交容量 → **用量条 x/上限**
- 外交对象列表 → 每项加 **态度色点**
- 信任/威胁/战略利益 → **量表 ×3**（威胁红/信任绿）
- 从属 自主权/忠诚度 → **量表**

### 军事
- 战争疲惫 → **量表(红)**
- 现役军团 → **军团卡：兵力条 + 士气/组织/补给三迷你条**
- 战争 → **战争卡 + 战争分数进度条**

### 发展
- 时代进度 → **进度条**；时代使命 → **检查清单**；导师指引 → **步骤点阵**
- 科技 ×9 → **门槛清单(年代/传播≥25%/思想点) + 采纳钮**

## 三、落地顺序
1. 基元 `widgets.js` + `hifi-widgets.test.cjs` + CSS（基础，先合）。
2. 国家 → 经济 → 外交 → 军事 → 发展 逐抽屉接线，每步 bump `drawers.js?v`、跑回归。
