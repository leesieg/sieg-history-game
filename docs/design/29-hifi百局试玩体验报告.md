# hifi 百局试玩体验报告

## 结论

当前版本不是单纯的数值微调问题，优先要修局势系统的状态机。

100 次开局尝试中，92 局跑满 40 季，8 局中途崩溃。另跑 5 个 100 季长样本，5 个全部崩溃。崩溃原因都和局势自动宣战有关：系统在已经交战或停战期内再次触发宣战。

| 项目 | 结果 |
|---|---|
| 测试对象 | `prototype/hifi/index.html` 当前 hifi demo |
| 批量试玩 | 100 次开局尝试 |
| 每局长度 | 40 季 |
| 完成局数 | 92 |
| 崩溃局数 | 8 |
| 长样本 | 5 个 100 季样本，全部崩溃 |
| 主要崩溃 | `双方已经交战`、`停战协定期内不能宣战` |
| 核心判断 | 先修状态机，再做平衡和交互优化 |

## 测试方式

本次测试分两部分。

| 类型 | 方式 | 目的 |
|---|---|---|
| 引擎层批量试玩 | Node VM 加载 hifi 同一套引擎脚本和地图种子数据，自动推进季度 | 看数值趋势、机制稳定性、长期崩溃 |
| 浏览器按钮巡检 | 浏览器加载 hifi 页面，逐类点击按钮、抽屉、tab、地图工具 | 看界面按钮是否有效、分类是否合理、是否有无效展示 |

> [!warning]
> 一开始尝试用完整浏览器脚本跑 100 局，但单次工具调用超过 5 分钟上限。最终改为引擎层跑 100 局，浏览器层单独做按钮巡检。这样能覆盖机制和界面，但不是人工逐手操作的体验测试。

### 数值问题

| 问题 | 表现 | 建议 |
|---|---|---|
| 合法性过低 | 25% 完成局合法性为 0 | 降低灾害、战争、改革对合法性的叠加惩罚；增加恢复手段 |
| 思想点固定 | 92 局全是 60 | 思想点应来自城市、贸易、科技传播、事件，不应固定收束 |
| 战争疲惫断裂 | 大多数为 0，少数直接 100 | 战争疲惫应按战斗、占领、封锁、补给逐季累积 |
| 国家贫富差过大 | 金钱最小 0，最大 3096 | 需要维护费、行政成本、贸易风险消耗强国盈余 |
| 非军事路线和军队脱节 | 大多数路线军团数为 0 | 外交、经济、局势目标也应要求最低防卫或军队存在 |

## 机制合理性

### 硬错误

| 问题 | 结果 | 必须修复 |
|---|---|---|
| 已交战仍自动宣战 | 100 局中 5 次崩溃 | 自动宣战前必须检查双方是否已经处于战争 |
| 停战期自动宣战 | 100 局中 3 次崩溃，长样本 5 次全崩 | 自动宣战前必须检查停战协定 |
| 长期样本无法跑满 | 5 个 100 季样本全部崩溃 | 中长期稳定性未达标 |

### 按钮与机制评估

| 模块 | 按钮/入口 | 当前表现 | 评价 |
|---|---|---|---|
| 国家详情 | 左上统治者、切换国家、确认国家 | 可打开、可切换 | 有效，但确认当前国家反馈弱 |
| 结束季度 | 顶部结束季度 | 可推进并弹季报 | 有效，但显示“问题 4”仍可推进，语义混乱 |
| 御前会议 | 右侧待办、标题 | 可打开会议 | 有效，但所有待办都导向会议，缺少精准跳转 |
| 局势 | 右下局势浮窗 | 可打开局势面板 | 入口有效，但底层状态机不稳定 |
| 国家-概览 | 展示政体、王权、统治者、合法性 | 纯展示 | 合理 |
| 国家-政制 | 改革、法律 | 可点击 | 条件和成本不够清楚，容易像免费按钮 |
| 国家-议会 | 议会未解锁、阶层分布 | 纯展示 | 合理，但未解锁状态下信息量偏少 |
| 国家-决议 | 国家决议、领土整合 | 可点击 | 条件不足的决议仍显示可点，体验不好 |
| 经济-财政 | 贸易政策、关税、敕令、议程 | 可点击 | 有效，但“当前关税”仍可点，应禁用 |
| 经济-贸易 | 商路投资 | 可点击 | 未解锁商路仍可点，应禁用 |
| 经济-建设 | 建筑、资本开发 | 部分可点，已建/不足会禁用 | 这一块较好 |
| 外交-邦交 | 国家列表、元首外交、宣战 | 可点击 | 功能多，但国家列表过长 |
| 外交-条约 | 贸易、通行、联姻、互不侵犯、同盟 | 可点击或禁用 | 需要展示成功率、原因、对方态度影响 |
| 外交-从属 | 朝贡、附庸、傀儡 | 可点击 | 对强国也能直接点，条件不足 |
| 军事-概览 | 军事点、战争疲惫、军团数、战争数 | 纯展示 | 合理 |
| 军事-军团 | 动员、雇佣、打开军团 | 可点击 | 反馈清楚，是目前较好的模块 |
| 军事-战争 | 提议停战 | 可点击 | 缺战分、目标、占领、损失细节 |
| 发展-概览 | 思想点、时代进度、使命、导师指引 | 纯展示 | 合理，但中期目标弱 |
| 发展-科技 | 科技采纳 | 可点击 | 条件展示清楚，但不满足条件仍可点 |
| 地图模式 | 政治、政体、家族、阶层、宗教、地形、人口、产出、贸易、军事 | 可切换 | 有效 |
| 地图工具 | 图例、缩略图、缩放 | 可点击 | 有效，但右下空间紧张 |
| 地块详情 | 关闭、展开、地块行动 | 可点击 | 有效，但和弹窗叠加时容易被遮挡 |

## 界面交互合理性

| 问题 | 表现 | 影响 | 建议 |
|---|---|---|---|
| 弹窗不互斥 | 季报、局势、御前会议可以同时打开 | 画面混乱，玩家不知道当前焦点 | 打开新 modal 前关闭其他 modal |
| 待办跳转不精准 | 战争、外交、经济待办都进入御前会议 | 玩家多点一步，还要自己找入口 | 待办直接打开对应抽屉和 tab |
| 外交对象过长 | 邦交页一次列出大量国家 | 扫描成本高 | 按邻国、强敌、盟友、附属、远方国家分组 |
| 假按钮太多 | 未解锁科技、未解锁商路、条件不足决议仍可点 | 玩家以为可执行，点击后才知道不行 | 条件不满足直接禁用，并展示缺口 |
| 结束季度文案不清 | 有问题时显示“问题 4”，但仍可推进 | 玩家误以为必须处理 | 改成“结束季度”，问题数量单独展示 |
| 右下控件拥挤 | 地图模式、图例、缩放、局势都在右下 | 小屏容易挤压 | 局势浮窗上移或折叠，地图工具压缩 |
| 资源来源不透明 | 顶部显示资源和趋势，但不解释来源 | 玩家难以判断为什么涨跌 | 点击资源打开季度账本 |
| 导师指引滞后 | 中期仍显示“选择一个地块” | 目标感弱 | 根据国家、时代、局势刷新主线目标 |

## 待办优先级

| 顺序 | 任务 | 原因 |
|---:|---|---|
| 1 | 修复局势自动宣战状态机 | 当前会直接崩溃，必须先修 |
| 2 | 所有条件不足按钮改为禁用 | 减少假操作，提高可读性 |
| 3 | modal 弹窗互斥 | 解决界面叠加混乱 |
| 4 | 待办直达对应系统 | 提升操作效率和目标感 |
| 5 | 调整合法性、思想点、战争疲惫曲线 | 解决数值断裂 |
| 6 | 外交对象分组 | 降低外交界面噪声 |
| 7 | 资源账本入口 | 让玩家理解经济变化 |
