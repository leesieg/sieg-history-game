# 44-共主邦联与继承合并

> 解 `42` 审计 C2：联姻→继承(38) × succession 模块(36) × 帝国选举(41) 三者交叉，但**靠联姻继承累积王冠的「共主邦联」没设计**——而这是近代早期头号机制，也**直接撞游戏论题「帝国的代价」**。
> 关键复用：共主邦联 = `41` 可复用超国家结构原语的**第四种类型 `dynastic`**（头=共君、成员=被联合的诸王国、向心力=替代权威）。
> 日期：2026-06-27

---

# 第一部分 · 设计总述

## A. 现状的病

- 联姻(38)只给一点信任/王朝纽带，**继承、共主邦联完全没有**。
- 而**哈布斯堡靠联姻继承累积勃艮第/西班牙/波西米亚/匈牙利**是近代早期的核心剧本——缺它，地图上的大国成长逻辑不真实。
- **庞大共主邦联极不稳定**（查理五世的帝国治不动、终被拆分）——这**就是「帝国的代价」**，缺它，游戏失去最切题的反雪球。

## B. 核心思想

> **联姻铺垫 → 继承/选举形成共主邦联 → 向心力拉扯（越大越杂越难 held）→ 整合为统一国家 或 解体。** 这条线就是「复合君主国 → 近代统一国家」的史实路径，也是游戏论题的可玩化。

- 共主邦联 = `41` 原语的 `dynastic` 实例：**头=共君、成员=诸王国、向心力(cohesion)替代权威**。
- 从邦**复用从属/神罗成员那套半自治实体**——保留自己的国家，只是共君。

## C. 关键设计决策（讨论结论，已定·全做）

- **C1. 全做**：形成 + 运作 + 向心力不稳定 + 整合 + 解体 + 继承战。
- **C2. 三条形成路径**：继承（绝嗣+宣称）/ 选举（选举制王位）/ 联姻铺垫。
- **C3. 从邦复用半自治实体**：保留自身政体/经济/阶层/军队(36/37)，共君、外交绑定、可征召。
- **C4. 整合路径**：从邦可随时间+向心力+行政能力**整合为核心**（通向统一近代国家）。

## D. 接缝 + 回到核心循环

> 联姻(38)→王朝宣称 →（绝嗣/选举触发）继承 → 共主邦联(41 dynastic 实例)→ 向心力受宗教(40)/差异/分离拉扯 → 整合(36 集权/39 官僚)或解体 → 反作用回版图与论题 → 循环。

# 第二部分 · 落地蓝图

## 1. 数据结构（41 原语的 dynastic 实例）

```js
// world.structures[`union:<senior>`] type:"dynastic"
union = {
  type:      "dynastic",
  head:      "西班牙",                 // 主邦：共君所在的资深王国
  members:   ["奥地利","波西米亚"],     // 从邦
  cohesion:  60,                       // 向心力（替代权威，0-100）
  sharedRulerOf: ["西班牙","奥地利","波西米亚"], // 共君统治的全部王国
};
// 每个从邦国家上：country.union = { senior:"西班牙", junior:true }
```

## 2. 形成（三路径）

| 路径 | 触发 | 结果 |
|---|---|---|
| **继承** | 某君主制(hereditary)君主**绝嗣/继承危机** + 外君有 **王朝纽带 + 宣称**(38) | 外君继承 → 该国成从邦、共君；竞争继承 → **继承战**(38 succession CB) |
| **选举** | 选举制王位（神罗/波兰）选出**外国君主** | 该位与外君本国结共主（雅盖隆式） |
| **联姻铺垫** | 联姻(38) → 王朝纽带 → 随对方王朝衰弱积累宣称 | 为上两条铺路 |

> succession 模块(36)决定谁能继承/被继承：**仅 hereditary/elective 君主制**可继承，共和制不可。

## 3. 运作

- 从邦 = 半自治实体（复用从属/神罗成员，38/41）：自有政体/经济/阶层/军队，但**共君、外交绑定主邦、可被征召、不可独立宣战**。
- **共君三维(36)惠及全邦联**：明君利整个邦联；幼主/摄政则削弱（拉低向心力）。

## 4. 向心力 / 不稳定（= 论题核心）

```
每季 cohesion 漂移：
  − 宗教差异(40)：从邦信仰 ≠ 主邦 → 重罚（天主西班牙 vs 新教尼德兰 = 尼德兰起义）
  − 文化/制度差异：政体原型/阶层结构不同
  − 地理分离：与主邦不接壤
  − 从邦实力：强从邦更不驯
  − 主君：幼主/弱君 → 低向心力
  + 共君外交三维、时间、整合进度、联姻纽带、共同敌人
```
**低向心力后果**：
- 从邦阶层闹**自治**（阶层不满，36）。
- **独立叛乱**：从邦脱离（独立战，38 防御/独立）。
- **继承解体**：下次继承时邦联分裂。

> **越大越杂越难 held**——这就是「帝国的代价」，也是天然反雪球：你无法靠联姻继承一路滚到统治世界而不撕裂。

## 5. 整合（C4：复合君主国 → 统一国家）

- 从邦 → 核心：需 **高向心力 + 行政能力（官僚科技 39 / 集权制度 36） + 时间 + 投入（行政点/钱）**。
- 缓慢把从邦转为**核心领土**（完全并入主邦的地块/阶层，走 36 reconcileEstates）。
- **这是复合君主国走向统一近代国家的路径**（接 40/41/36 近代国家）。
- 整合期从邦阶层抵抗（阶层不满）。

## 6. 解体

- **分割继承**：继承时嗣子分裂（查理五世 → 西班牙系 + 奥地利系）→ 邦联裂成多个继承邦联。
- **向心力崩** → 独立叛乱 → 邦联缩水/瓦解。

## 7. 神罗交互（41）

- 哈布斯堡 = 皇帝(41) + 靠共主累积王冠(44) = 最强复合君主国。
- 神罗**选举可形成共主**（选外君为帝）。
- 一个神罗成员可与非神罗王国结共主（奥地利[神罗] + 西班牙[非神罗]）——两套超国家结构叠加。

## 8. 接缝

| 接缝 | 机制 |
|---|---|
| 共主邦联 = 41 原语 dynastic 实例 | head=共君、members=诸国、cohesion=向心力 |
| 联姻→宣称→继承/继承战 | doc 38 |
| 谁能继承/被继承、从邦保留制度、整合=集权 | doc 36 succession 模块 + reconcileEstates |
| 宗教差异拉低向心力 | doc 40（天主君主统治新教臣民=宗改张力） |
| 神罗选举形成共主、哈布斯堡累积 | doc 41 |
| 向心力崩=反雪球，整合=近代统一国家 | 游戏论题 |

## 9. 消费方迁移

| 文件 | 改动 |
|---|---|
| `engine/structures.js`(41) | 加 `dynastic` 类型：共主形成/向心力/整合/解体；复用原语 head/members |
| `politics.js`(36) | 继承事件检外国宣称者 → 触发继承；从邦保留自身制度；整合=从邦转核心 |
| `diplomacy.js`(38) | 联姻→王朝宣称累积；继承战 CB；共主外交绑定 |
| `history.js` | 向心力漂移；继承时分割判定；独立叛乱触发 |
| `strategy.js` | AI 联姻/继承策略；AI 管邦联（整合 vs 维持）；从邦 AI（不驯/叛乱），带默认 getter |
| `drawers.js`/`dialogs.js` | 共主邦联面板（成员/向心力/整合进度/共君）、继承事件弹窗 |

## 10. 分阶段落地

| 阶段 | 内容 | 依赖 |
|---|---|---|
| **形成** | 联姻→宣称→继承/选举形成共主（41 dynastic 实例） | doc 36(succession) + 38(联姻/宣称) + 41(原语) |
| **运作 + 向心力** | 从邦半自治、共君三维、向心力漂移 | 随上 |
| **整合 + 解体** | 从邦→核心；分割继承/独立叛乱 | 随上 + 39(官僚科技) |
| **数值调平** | 向心力升降率、整合成本、叛乱阈值 | **sim 之后** |

> 共主是**长局/论题特性**（哈布斯堡式累积在中后期兑现），优先级低于核心循环；但它定义了「帝国的代价」的最切题玩法。

## 11. 风险

| 风险 | 缓解 |
|---|---|
| 联姻继承滚雪球吞并全图 | 向心力不稳定是刹车；分割继承；独立叛乱；**越大越难 held** |
| 机制复杂 | 复用 41 原语 + 38 从属机器，不另起炉灶 |
| AI 管邦联/从邦叛乱 | 启发式（整合 vs 维持；从邦按向心力叛乱）；带默认 getter 防崩 |
| 继承随机性挫败 | 继承走宣称+绝嗣的**确定性**判定，非纯随机；玩家可经营联姻铺路 |
| 与神罗双重结构叠加复杂 | 两套超国家结构共用原语；UI 分层展示 |
