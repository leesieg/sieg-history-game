# 05 - 国家外交系统开发路线图

结论：先完成双边外交闭环，再扩展从属关系，最后开发完整国际组织。每个阶段都必须能独立游玩和验证。

## 总体路线

```mermaid
flowchart LR
    A["阶段一<br>关系底层"] --> B["阶段二<br>外交操作"]
    B --> C["阶段三<br>契约与从属"]
    C --> D["阶段四<br>战争与和平"]
    D --> E["阶段五<br>完整国际组织"]
```

| 阶段 | 目标 | 主要交付 |
|---|---|---|
| 一 | 让国家关系可计算、可解释 | 双边关系、动态战争、国家态度、外交资源 |
| 二 | 让玩家可以主动改变关系 | 使节任务、领导人外交、确定性提案 |
| 三 | 让承诺和支配关系产生实际效果 | 条约生命周期、外交容量、朝贡/附庸/傀儡 |
| 四 | 让外交通向战争和和平 | 战争借口、参战义务、和平谈判、停战 |
| 五 | 让多国共同制度成为玩法 | 组织选举、议案、改革、执法、成员战争规则 |

## 阶段一：关系底层

| 内容 | 完成标准 |
|---|---|
| 双边关系表 | 每对国家分别保存信任、威胁、恩义、领土矛盾、制度冲突和战略利益 |
| 国家态度 | 自动生成亲近、合作、中立、警惕、竞争、敌对 |
| 动态战争 | 删除固定敌对名单，战争与停战进入世界状态 |
| 外交资源 | 国家拥有使节和外交容量；外交点沿用领导人能力产出 |
| 历史事实层 | 战争、条约、从属和组织身份与模拟关系数值分开保存 |

## 阶段二：外交操作

| 内容 | 完成标准 |
|---|---|
| 改善关系 | 派遣使节后，每季度提高信任、降低威胁 |
| 领导人会晤 | 改变友谊、尊重等私人关系 |
| 赠礼与威慑 | 分别改善关系或增加恐惧、威胁和宿怨 |
| 外交提案 | 玩家能看到接受意愿的每项修正和最终结果 |
| 外交界面 | 国家详情与左侧外交抽屉都能查看和发起行动 |

## 阶段三：契约与从属

| 内容 | 完成标准 |
|---|---|
| 平等契约 | 贸易协定、军事通行、王室联姻、互不侵犯、防御同盟 |
| 生命周期 | 契约有期限、退出规则、违约惩罚和季度履约 |
| 外交容量 | 长期承诺占用容量，超额会降低信任和外交效率 |
| 从属类型 | 支持朝贡国、附庸国、傀儡国 |
| 从属条款 | 记录外交、宣战、军事、贡赋、市场、继承权限 |
| 自主与忠诚 | 法定自主权和现实忠诚度分别结算 |

## 阶段四：战争与和平

| 内容 | 完成标准 |
|---|---|
| 战争借口 | 领土、违约、独立、保护和组织执法产生不同借口 |
| 盟友参战 | 根据契约义务、利益和领导人态度响应 |
| 战争目标 | 领土、贡赋、从属、条约与组织目标分开结算 |
| 和平谈判 | 战争分数限制条件，不允许无成本索取 |
| 停战与违约 | 和约形成停战，撕毁停战产生严重外交后果 |

## 阶段五：完整国际组织

| 内容 | 说明 |
|---|---|
| 多边成员体系 | 成员、职位、权利和义务 |
| 组织选举 | 皇帝、领袖或议长的选举 |
| 议案与投票 | 成员根据利益、关系和国内政治表决 |
| 组织改革 | 改变成员自治、共同防务和中央权威 |
| 组织执法 | 调停、禁令、非法战争和领土归还 |
| 成员战争规则 | 组织法限制成员之间的宣战和参战 |

阶段五开发前，必须先验证阶段一至四能稳定处理国家关系、条约义务和动态战争，否则多边规则无法可靠运行。

## 当前 Demo 开发范围

本轮开发完成阶段一至三的可玩闭环，并为阶段五保留组织数据结构和身份展示。

| 本轮开发 | 本轮不开发 |
|---|---|
| 双边关系与自动态度 | 组织议案与投票 |
| 动态战争状态 | 组织改革路线 |
| 使节、外交容量 | 皇帝或组织领袖选举 |
| 领导人外交 | 组织禁令与执法 |
| 确定性外交提案 | 成员战争法 |
| 平等契约及履约 | 完整战争与和平谈判 |
| 朝贡、附庸、傀儡 | 秘密外交和复杂间谍系统 |
| 组织身份底层与展示 | 全量历史外交数据库 |

## 验收节点

| 节点 | 验收问题 |
|---|---|
| 关系闭环 | 为什么两国是竞争或合作，玩家能否看懂？ |
| 行动闭环 | 一次外交行动是否消耗资源并改变季度局势？ |
| 契约闭环 | 条约是否真实改变收益、通行、义务或容量？ |
| 从属闭环 | 宗主是否获益，附属国是否会因压力降低忠诚？ |
| 换代闭环 | 领导人关系是否变化，而国家契约仍然保留？ |
| 切国闭环 | 切换国家后，双方看到的契约与方向性关系是否一致？ |
