## 一、范围与分期

| Phase | 主题 | 覆盖待办 | 层 |
|---|---|---|---|
| **A — 数值闭环** | 资源消耗出口（维护费基线 + 事件脉冲）+ 季报展示净额 | #4 | engine + 测试 |
| **B — 反馈链** | 每个自由动作显示成本 / 预期收益 / 下季影响，执行后 toast 确认 | #5 | engine + ui |

## 二、Phase A — 资源消耗出口（#4）

核心：`economy.js` 的 `settleCountry` 在产出结算后**扣维护费**得到净额；季报 `quarterLedger` 展示「产出 − 维护 = 净增」三段。让"什么都不做"的玩家净增接近零或微正，扩军 / 铺建筑把净额压成负，逼出取舍。

### 2.1 维护费（基线、可规划）

在 `settleCountry` 产出累加之后、写回 state 之前，计算各资源维护：

| 资源 | 维护出口 | 公式方向 |
|---|---|---|
| 粮食 food | 军团口粮 | 全国军团总兵力 × 单兵口粮系数 |
| 军需 military | 常备/职业军维护 + 炮兵 | 按 unit.serviceType 分档：常备/职业高、征召低、卫队免 |
| 金钱 money | 建筑行政维护 + 佣兵饷银 | 己方建筑数 × 维护系数 +（佣兵欠饷已有逻辑，接入账本） |

- 系数集中定义为 `economy.js` 顶部常量（`MAINTENANCE`），便于标定与测试。
- **欠费惩罚走已有压力通道，不新建系统**：
  - 粮食为负 → 复用 warfare 补给/士气下降路径（军团补给扣减）。
  - 军需为负 → 常备军组织度下降（warfare 已有 organization 字段）。
  - 金钱为负 → 复用 politics 合法性 / 阶层满意压力。
  这样消耗回灌核心循环，符合 19 号"双向调节阀"范式，不是空转扣数。

### 2.2 事件消耗（脉冲、不可预期）

- 扩展已有 `situations`（黑死病第 12 季 / 小冰期第 48 季）爆发期：除扣人口外，追加扣粮 / 钱（救济成本），写进 `quarterLedger` 的"事件"段。
- **不新增随机事件系统** —— 只在现有 situation 框架上挂消耗（YAGNI）。

### 2.3 季报展示

`quarterLedger`（2 期已存在）每资源新增"维护"与"事件"两段，结构改为：

```
{ resource, gross（产出）, maintenance（维护）, event（事件消耗）, net（净增） }
```

季报 UI 渲染三段构成；net 为负时高亮提示。

### 2.4 测试

- `hifi-economy`：新增维护费结算用例（造一支大军团 → food 维护 > 0；建多栋建筑 → money 维护 > 0；net = gross − maintenance − event）。
- `hifi-longrun`：新增"资源不膨胀"断言 —— 在持续扩军场景下，连续 N 季资源净额转负 / 不再单调暴涨。
- `hifi-warfare` / `hifi-politics`：欠费惩罚通道断言（food 负 → 补给降；money 负 → 合法性压力）。
- `hifi-demo2-parity`：按 §一 风险结论处理。

## 三、Phase B — 操作反馈链（#5）

目标：每个**自由动作**（花行动点的：建设 / 敕令 / 法律 / 改革 / 动员 / 外交提案 / 整合）执行前显示「成本 + 预期收益 + 下季影响」，执行后 toast 确认实际变化。复用 `feat/hifi-objectives-council` 已建的 `proposals.js` 校验与效果预览能力，下沉到抽屉里每个动作按钮。

### 3.1 预览纯函数

为每类动作提供 `previewEffect(world, action) → { 资源: "+25/季", 合法性: "-2", ... }`：

- 纯函数预测，与引擎真正结算分开实现，但**口径必须一致**：预览说 +25，结算就该 +25。
- 渲染在动作按钮下方小字。
- 成本格式统一为 `成本: 24金钱 + 1行政点`。

### 3.2 校验与置灰

- 不可达 / 不可负担的动作，预览直接显示失败原因（复用 proposals 的 validate 失败文案），按钮置灰 —— 与御前会议草案卡一致。

### 3.3 执行后确认

- 动作执行后比对 state 差值，toast「鲁昂已建市场，下季金钱 +X」。

### 3.4 测试

- `hifi-economy` / `hifi-politics` / `hifi-diplomacy`：各加"预览值 == 实际执行后差值"断言（口径一致是硬约束）。
- `hifi-ui-smoke`：正则核对按钮下方预览容器存在、失败态置灰类存在。

## 四、Phase C — 界面减负（纯 UI）

只动 `ui/` + `index.html`，引擎不变。

### 4.1 #6 HUD 顶栏

资源栏**不折叠，保留 7 个**。改动：

1. **去掉游戏名称**：删除 `.game-title`「帝国的代价」。保留 `#countryShield` 盾徽（身份信息）。`brand-block` 收成只剩盾徽。
2. **时间/回合左挪、中间留白**：`.chronicle`（`#dateMain` + `#dateEra`）紧贴盾徽左对齐；顶栏 layout 改为 `左: 盾徽+日期 | 中: 弹性留白 | 右: 资源栏`，中间不放东西。
3. **去掉「待办」按钮、并入「问题与对象」**：删除 `#topPending`（`待办 N ›`）。计数与入口合并进右侧 `#issuePanel`：`issue-heading` 标题带计数（「问题与对象 · N」）；空状态折叠为小徽章。底部 `#seasonText`「处理待办 N」改为读 issue 队列。
4. **资源栏压缩**：`.resource-strip` token 间距/字号收窄；窄屏（横屏手机）下 `resource-token` 用 `flex-wrap` + 固定宽度换行成**两栏**，纵向两行而非横向溢出。

### 4.2 #10 地图模式盘

右下地图工具：只显示**当前模式**名 + 图标，点击展开模式盘；模式盘按 22 号 §6.3 二级分组：

| 一级 | 二级 |
|---|---|
| 政治 | 国家、政体、家族、阶层 |
| 地理 | 地形、气候、控制力 |
| 经济 | POP、产出、贸易品、市场 |
| 军事 | 军团、占领、补给、战争目标 |

小地图默认收起。（注：现有透镜以现存的 10 个为准，分组只重排不强行新增二级未实现的透镜；未实现项不显示。）

### 4.3 #11 地块面板折叠

左下 `#provincePanel` 默认迷你卡（地名 + POP + 1 行关键值），点击展开完整 `province-data`。

### 4.4 #7 地块动作按归属

`map.js` `updateProvince` 渲染动作前判断 `tile.owner` vs `playerCountry`，抽出纯函数 `tileActionsFor(tile, player) → 动作集`：

| 地块类型 | 动作 |
|---|---|
| 己方 | 建设 / 征兵 / 整合 / 派驻 |
| 外国和平 | 查看国家 / 外交 / 宣战 / 标记目标 |
| 外国交战 | 进军 / 围城 / 查看战争目标 |
| 海域（无海军） | 只读 |

### 4.5 #9 军令文字

军团面板按钮由纯图标改为图标 + 短文字：拆分 / 合并 / 补员 / 训练 / 复员 / 领军 / 规划路线。

### 4.6 #6 外交列表重构

外交抽屉对象列表按 `敌国 > 邻国 > 可缔约 > 其余` 排序；顶部加搜索框筛选；默认选中对象改为优先敌国 / 战争相关国（取代固定阿拉贡）。

### 4.7 测试

- `hifi-layout` / `hifi-ui-smoke`：去掉对 `.game-title` / `#topPending` 的核对；新增 issue-heading 计数、资源两栏容器、模式盘容器、迷你卡、军令文字按钮的核对。
- `hifi-map`：新增 `tileActionsFor(tile, player)` 纯逻辑断言（己方 / 外国和平 / 外国交战 / 海域 各返回正确动作集）。
