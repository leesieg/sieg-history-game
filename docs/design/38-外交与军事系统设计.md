# 38-外交与军事系统设计（合订）

> 承接 `36`（国家系统）、`37`（经济系统）后的设计讨论结论。外交与军事**不是两次独立重构，而是同一个闭环**——故合订。
> 核心环：**宣称 → 战争 → 和约 → 声誉 → 回灌关系**。两个接缝挂进已梳理系统：**军事制度模块（doc 36）= 国家↔军事**、**经济物产（doc 37）= 军事↔经济**。
> 日期：2026-06-27

---

# 第一部分 · 设计总述

## A. 现状的病

**外交（diplomacy.js）**
| 现状 | 病 |
|---|---|
| 除停战期/已交战，**对谁都能随便宣战** | 战争无理由、无差异化目标 → 报告 33「局势长期 open_war、战争浅」的根 |
| 关系是一堆孤立标量，主要靠战争/使节推动 | 不接利益网：共同敌人、贸易依赖、王朝纽带、声誉都没真正驱动 |
| 毁约/背盟**零代价** | 外交承诺不值钱 |
| 与经济/宗教脱节 | 有了 30 物产+禁运（37）、宗教剥离（36），外交本该长出经济战、继承宣称 |

**军事（warfare.js）**
> 军事是**现有最成熟的系统**（报告：稳定、测试覆盖好），战斗解算不浅（地形/兵种/将领/士气-组织-补给/经验）。**病不在「浅」，在「孤立」**：
| 现状 | 病 |
|---|---|
| 动员只扣抽象人口+军需 | 没接 30 物产（铁/马/硝石/木材） |
| serviceType 逐单位设 | 没接军事制度模块 → 五原型打起来一个样 |
| 战争一个目标地块、分数**只来自占领**、和约非取地即停战 | 战争目标/分数/和约都浅 |

## B. 核心思想：一条环 + 两个接缝

```
关系流/王朝/贸易依赖/威胁（压力层）
        │ 生成
        ▼
     宣称 / 开战动机 ──▶ 战争（军事制度+物产 决定能打什么）
        ▲                      │ 占领/战斗/疲惫
        │ 回灌                  ▼
     声誉 / 关系 ◀──── 和约（按宣称目标 × 战争分数 结算 → 改基底）
```

- **外交**生成「关系态势 + 宣称」，**军事**执行，**和约**按目标结算，**声誉/关系**回灌——闭环。
- 接缝一：**军事制度模块**决定你能拉出什么军队（国家↔军事）。
- 接缝二：**经济物产**卡住兵源与兵种（军事↔经济）。

## C. 关键设计决策（讨论结论，已定）

### C1. 轻量 CB：宣称是「代价修正」，不是「开战触发器」

**开战动机永远来自需求与机会**（下表），有没有宣称只决定打得贵不贵、占得稳不稳、拿得多不多：

| 开战动机（永远存在） | 来源流 |
|---|---|
| 掠夺：缺铁/马/黄金/粮 → 打产地 | 经济物产（37） |
| 安全：崛起邻国威胁高 → 先发制人 | 关系·威胁感 |
| 机会：接壤弱邻+实力悬殊 | 实力差 |
| 霸权：逼附庸、拆对方集团 | 阵营/从属 |
| 王朝：联姻继承争议（本身即宣称） | 王朝纽带 |

> **有宣称 = 便宜、合法、占领区低不满；无宣称 = 照样能打，但掉声誉、疲惫高、占的地高不满难消化。** CB 不堵动机，只调代价——让战争从压力网**涌现**。

### C2. 军事制度模块**决定**（硬，非系数）你能打什么

军事制度（doc 36）**直接决定可用兵种/兵源/维护**，不是 ×1.2 修正：封建征召拉不出常备军、撑不起长期征服；全民皆兵海量廉价军；雇佣灵活但忠诚低；常备军精锐但烧钱。**能打哪种战，被政体卡死。**

### C3. 关系三层：纽带类型 / 驱动维度 / 态度，新增声誉流与利益网

关系不再几乎只由实力差算，而从「共同敌人、贸易依赖、王朝纽带、声誉」这张利益网长出（§1）。

### C4. 战争多目标：分数纳入战斗+占领+疲惫，和约按目标结算

战争类型由 CB/目标决定（征服/附庸/劫掠/羞辱/继承/防御），和约 = f(目标, 战争分数)；分数补入**战斗胜负**（现状只占领计分）。

### C5. 禁运：经济外交杆（接 37）

一国可对另一国禁运，切断两国买卖，缺方更疼——经济战手段，与停战/同盟并列。

## D. 不重写战斗数学

`warfare.js` 的战斗解算（`resolveBattle/sidePower/applyCasualties/advanceOccupation`）是成熟资产，**本方案不动它**，只在其上加：宣称/战争目标层、分数补战斗、兵源接物产/制度、和约多样化。

# 第二部分 · 落地蓝图

## 1. 国家关系

分三层：**纽带类型（结构）/ 驱动维度（流）/ 态度（派生）**。

### 1.1 纽带类型（玩家能签/能撕）
| 纽带 | 现状 | 改动 |
|---|---|---|
| 无约/中立 | ✓ | — |
| 条约 | 贸易/通行/联姻/互不侵犯/同盟 | 联姻→产生**王朝宣称**（新） |
| 从属 | 朝贡/附庸/傀儡 | 沿用（P1-1 已激活） |
| 敌对 | 交战/世仇/竞争 | — |
| **禁运** | — | 新增（§经济战） |
| 阵营/集团 | 同盟自动参战雏形 | 涌现化、可见化（§9） |

### 1.2 驱动维度（受什么影响 / 影响什么）
| 维度 | 受什么影响 | 影响什么 |
|---|---|---|
| 信任 trust | 守约/赠礼/联姻/并肩作战↑；背盟/威慑/毁约↓ | 缔约·从属接受度 |
| 威胁感 threat | 对方实力、陈兵边境、扩张 | 安全型开战、拒绝臣服 |
| 战略利益 strategicInterest | **共同敌人、贸易依赖、地缘**（新增前二） | 同盟意愿、拉拢价值 |
| 领土矛盾 territorialConflict | 重叠宣称、边境、近期战争 | 领土型 CB、复仇 |
| 制度/信仰亲和 | 政体原型相近；信仰立场（→**信仰子系统**回灌） | 阵营归边 |
| 王朝纽带 kinship | 联姻 | **继承宣称 / 共主邦联** |
| **声誉 reputation**（新，国家级） | 守约↑ / 毁约背盟/无CB战↓ | 全局缔约意愿（§2） |

> 三条关键新驱动：**贸易依赖**（经济接缝）、**王朝纽带→宣称**（外交↔军事接缝）、**声誉流**。

### 1.3 态度（派生展示）
`亲密/合作/中立/戒备/竞争/敌视` 由上述维度聚合（沿用 `diplomaticAttitude`，输入变丰富），用于外交列表分组与 AI 取向。

## 2. 声誉系统（新流）

```js
country.reputation = 60; // 0-100，国家级
// ↑：和平守约每季缓升；履行同盟义务
// ↓：撕毁条约/停战、对有互不侵犯/同盟者背刺、无 CB 开战
```
- **影响**：`evaluateProposal` 加「声誉」项——低声誉者他国拒绝缔约/同盟；AI 对低声誉国戒备、不结盟。
- 把「外交承诺」变成有价资产，治「毁约零代价」。

## 3. 宣称 / CB（轻量）

### 3.1 宣称记录
```js
country.claims = [ { target:"英格兰王国", type:"dynastic", tileId?, region?, sinceTurn } ];
```
| 宣称类型 | 来源 |
|---|---|
| 王朝 dynastic | 联姻→王朝纽带，随时间产生继承宣称 |
| 领土 territorial | 接壤/历史归属 |
| 霸权 hegemonic | 高威胁/竞争对手 |
| 捏造 fabricated | 花外交点+时间制造 |

### 3.2 代价修正（轻量 CB 的核心）
开战时匹配是否持有对应宣称：

| | 有匹配宣称 | 无宣称 |
|---|---|---|
| 战争疲惫累积 | 低 | 高 |
| 声誉损失 | 几乎无 | 明显↓ |
| 占领区不满/整合难度 | 低（合法征服） | 高（强占难消化） |
| 和约可索取 | 多（达成目标） | 受限/更贵 |

- 实现：`declareWar(world, attacker, defender, goal)` 增 `goal` 与 `cbMatched` 标记；`canDeclareWar` 仍只硬挡停战期/已交战（不堵动机）。

## 4. 战争类型与触发

### 4.1 类型（决定 war.goal 与和约结算）
| 类型 | 目标 | 典型 CB |
|---|---|---|
| 征服战 conquest | 索地块/区域→兼并 | 领土/王朝 |
| 附庸战 subjugation | 强制朝贡/附庸 | 霸权/威胁 |
| 劫掠战 plunder | 不兼并，抢资源/黄金、破坏 | 无需 CB |
| 羞辱/霸权战 humiliation | 压合法性、拆集团、逼放附属 | 竞争 |
| 继承/共主战 succession | 夺王位→继承/共主邦联 | 王朝纽带 |
| 防御/独立战 defensive | 防守方 / 附属求独立 | 自带 |
| （圣战，后续） | 信仰目标 | 信仰子系统 |

### 4.2 触发（= 动机，非 CB）
AI `warTarget` 打分：**收益（目标虚弱度 + 我缺的资源 + 其威胁 + 霸权价值） − 成本（疲惫 + 无 CB 声誉惩罚 + 军事制度可持续性）**。玩家同样按需求/机会自由发起，CB 影响代价。

## 5. 军事制度模块 → 可拉军队（接缝一，硬决定）

`mobilizeArmy` 读 `institutions.military`（doc 36）**决定**可征兵种/兵源/维护，非修正：

| 军事制度 | 可拉兵种/兵源 | 维护/特性 |
|---|---|---|
| 封建征召 | 征召步兵 + 贵族骑士；不能拉常备职业军 | 便宜起、弱持续、依赖贵族阶层 |
| 常备军 | 职业兵种 | 维护贵、质量高；需货币化财政+绝对主义纪元 |
| 全民皆兵 | 海量征召；人力上限×1.5 | 廉价、适合扩张/防御 |
| 雇佣立国 | 佣兵 | 灵活、忠诚低、欠饷叛离 |

> 这让五原型「打起来不一样」——军事版的「每个财政模块把钱的机器拧成不同形状」。

## 6. 兵源接经济物产（接缝二）

| 物产 | 军事用途 |
|---|---|
| 马 horses | **骑兵半硬闸**：无产地且被禁运 → 动员不出骑兵 |
| 铁/铜/锡 | 军需库存（养兵规模，软约束） |
| 硝石 saltpeter | 火炮经济（解锁/供养炮兵） |
| 木材/海军物资 | 造船/攻城器械 |

军团编制被「你产/能买到什么」卡住（买受禁运/战争影响）。

## 7. 战争中影响（持续流）

| 影响面 | 怎么变 |
|---|---|
| 战争疲惫 | ↑ → 合法性↓（沿用） |
| 占领 | 占领区零产出（经济失血）；占满→战争分数+25（沿用） |
| 战斗 | 伤亡、破坏↑、人口↓；**战斗胜负计入战争分数（新增，现状只占领计分）** |
| 后勤 | 军队吃占领区/本土粮；残破地养不起军（接 37 粮食） |
| 关系 | 交战双方领土矛盾+威胁逐季累积（沿用） |
| 经济 | 军需/粮消耗；战区航路成本↑；禁运 |
| 阶层 | 军事阶层（贵族/战士）满意↑ 或 平民被征↓ |
| 声誉 | 无 CB 战争持续掉声誉 |
| 同盟 | 盟友连带参战（集团战，§9） |

## 8. 战争结束 / 和约（按目标 × 分数结算）

| 影响面 | 怎么变 |
|---|---|
| 领土/附属/赔款 | 据 war.goal 转移领土 / 建立朝贡附庸 / 赔款贡赋 |
| 停战期 | 20 季不可再战（沿用 truces） |
| 合法性 | 胜方威望↑、败方↓ |
| 占领区消化 | **有 CB 征服→低不满；无 CB→高不满、难整合**（接 36 整合/合法性） |
| 关系/声誉 | 败方记仇（领土矛盾/世仇留存）；苛刻条款/无 CB → 声誉↓、他国戒备 |
| 阶层 | 胜负回灌军事阶层满意 |
| 疲惫 | 和平期逐季消退（沿用） |
| 纪元/局势 | 触发因果链（如君堡陷落，已有） |

> `concludePeace` 扩展：按 `war.goal` 类型给不同条款集（取地/附庸/赔款/放附属），分数门槛分级（现状仅 target_territory 需 score≥25）。

## 9. 阵营 / 集团（涌现）

- 同盟 + 从属网络 → 集团；战争把整块拉下水（现有同盟自动参战的扩展）。
- 关系驱动「共同敌人↑战略利益」自然把敌人的敌人拉成潜在盟友。
- UI：外交界面标注集团归属与连带风险。

## 10. 消费方迁移

| 文件 | 改动 |
|---|---|
| `diplomacy.js` | +`reputation`、+`claims`、`evaluateProposal` 加声誉项；`strategicInterest` 接共同敌人/贸易依赖；联姻→产生王朝宣称；+`embargo` 动作 |
| `warfare.js` | `declareWar` 加 `goal`/`cbMatched`；`mobilizeArmy` 读军事制度定兵种、读物产卡兵源；`resolveBattle` 战果计入 `war.score`；`concludePeace` 按 goal 多样化条款。**战斗数学不动** |
| `strategy.js` | `warTarget` 打分接「需求/安全/机会/霸权 − 疲惫/无CB声誉」；AI 用宣称、按军事制度可持续性决策；AI 禁运仅对敌对/战争国 |
| `history.js` | 声誉/疲惫回灌；无 CB 占领的高不满接整合/合法性 |
| `data/`（新） | `claims.js`/CB 与战争目标定义；或并入现有 |
| `drawers.js`/`dialogs.js` | 外交：声誉、宣称、集团、禁运可视；军事：战争目标、和约条款选择 |

## 11. 分阶段落地

| 阶段 | 内容 | 排序 |
|---|---|---|
| **声誉 + 关系利益网** | reputation 流、strategicInterest 接共同敌人/贸易依赖 | **加法，早做**（不卡 sim） |
| **宣称 / 轻量 CB** | claims、联姻→宣称、开战代价修正 | 加法，早做 |
| **战争目标 + 和约多样化** | war.goal、分数补战斗、concludePeace 按目标 | 加法，早做（战斗数学不动） |
| **禁运** | diplomacy.embargo（接 37 市场） | 随 37 层2 市场重写 |
| **军事制度→兵种**（接缝一） | mobilize 读 institutions.military 硬决定 | **依赖 doc 36 模块化** |
| **兵源接物产**（接缝二） | 马/硝石/木材 等卡编制 | **依赖 doc 37 层1** |
| **阵营/集团可视** | 集团涌现+UI | 加法 |

> 大部分（声誉/宣称/战争目标/关系）是**加法、可早做，不卡 sim**；只有两个接缝分别依赖 36/37。战斗数学全程不动。

## 12. 风险

| 风险 | 缓解 |
|---|---|
| 轻量 CB 仍让 AI 乱战 | `warTarget` 把「无 CB 声誉惩罚+疲惫」计入成本；AI 优先有宣称目标 |
| 声誉螺旋（一次毁约全图敌视） | 声誉缓变、缓恢复；惩罚分级（撕停战 < 背盟 < 无 CB 灭国） |
| 军事制度硬决定致弱国无军可用 | 每种制度都有可用下限军（封建至少有征召兵）；带默认 getter 防崩 |
| 禁运被 AI 滥用断全图贸易 | AI 禁运预算 + 仅对敌对/战争国 |
| 战争目标×和约组合爆炸 | 目标类型封顶 6 种，每种一套固定条款集 |
| 改 war 对象结构碰崩溃轨 | 复用 35 号/P1-2 的带默认守卫；invariants 5×200 守门 |
