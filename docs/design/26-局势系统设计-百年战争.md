# 26 - 局势系统设计（Struggle）与百年战争实例

> 设计先行文档。范围：MVP 局势内核 + 百年战争单实例。参考 CK3《十字军之王 3》局势（Struggle）系统。
> 上游讨论：见 `docs/plans/2026-06-22-france-campaign-loop.md`（法兰西战役体验闭环），本方案把该计划的 Phase 1/2/5「目标—战争—结算」收纳进统一的局势系统。

## 1. 背景与目标

试玩法兰西 1337 开局的核心问题是「主线压力不足」：百年战争只是外交页里一行「百年战争 / 提议停战」，没有压住全局，玩家不知道怎样赢。

把百年战争这类**影响多国、跨越数十年的特殊历史事件**抽象成一个**可交互的「局势」**：

- 所有参与国共享同一个局势对象，各自有对应的局势界面查看进展、操作、承受影响。
- 局势有**阶段节奏**（对峙→鏖战→议和→定局），靠**诱因事件**推动阶段流转，而非单调进度条。
- 局势**不能靠涂地图赢**，必须由某一统治者推「终局决议」终结，给永久区域修正。

目标：把零散的战争列表变成一个有阶段节奏、多国共享、有历史终局的舞台，直接增强历史代入感与主线压力。

## 2. CK3 局势内核映射（已核对）

| CK3 Struggle | 本项目「局势」 | 现有接缝 / 改动 |
|---|---|---|
| Struggle 对象 | `situation.type = "struggle"`，与现有被动情势（黑死病/小冰期）并存 | 复用 `world.situations[]`；新类型不进旧判定，守回归测试 |
| 地理区域 | `regionTiles`（争议地块集） | tiles 已有 `id`/`control` |
| Involved / Interloper / 旁观 | `involvement`：当事国 / 干涉者 / 旁观 | 按首都 + 争议地 + 交战方判定 |
| Phases | 阶段：对峙 → 鏖战 → 疲惫议和 → 定局 | 现 `phase` 单调（预兆/爆发/消退）换成博弈阶段 |
| Catalysts → 阶段计量条 | 诱因事件给阶段分加权，过阈翻阶段 | 现 `progress+5` 单调，改为多向计量 |
| 时间默认诱因（每年 +1） | 每季向「默认下一阶段」漂移 | `advanceQuarter` 已每季跑 `processSituations` |
| 阶段限定 interactions | 阶段限定操作（gate + 调现有引擎） | 接行动点 + warfare/diplomacy，不另造战斗 |
| Ending decisions（多终局） | 终局决议：法兰西霸权 / 英格兰主张得逞 / 谈判和平 / 长期僵局 | 复用「三段使命状态」做结算口径 |
| Struggle UI window | 局势界面：进展 / 操作 / 影响 | drawers/dialogs |

来源：CK3 Wiki – Struggles / Situation、Paradox Dev Diary #94「Anatomy of a Struggle」、PCGamesN 解析。

## 3. 与现有系统的关系（回归保护，关键约束）

现有 `world.situations[]` 装的是**被动情势**（黑死病、小冰期）：到点自动起、progress 单调 +5、三相按阈值、对所有国家一视同仁、玩家不能操作（`history.js:228-280`）。

**约束：局势（struggle）作为新类型，不改动被动情势的任何旧逻辑。**

- `startSituation` / `processSituations` 现有代码路径保持不变（黑死病/小冰期、爆发脉冲、`lastEffectTurn` 节流、`lastReport.event` 写法照旧）。
- 局势走**独立函数** `startStruggle` / `processStruggles`，在 `processHistory` 管线里与 `processSituations` 并列调用。
- 存量 hifi 测试（37 个）必须继续全绿；局势新增逻辑只加测试、不改旧断言。

## 4. 数据结构

```js
// world.struggles[]（与 world.situations[] 区分；或共用数组以 type 区分，二选一，倾向独立数组更清晰）
struggle = {
  key: "hundred_years_war",
  label: "百年战争",
  type: "struggle",
  startedTurn: 1,
  regionTiles: ["paris", "rouen", "orleans", "gascony", "aquitaine", "flanders", "calais"],
  // 参与国（数据驱动，按 CAMPAIGN_STAGES 同样思路可扩展到别的局势）
  parties: {
    france:  { role: "principal" },           // 当事国
    england: { role: "principal" },
    burgundy:{ role: "interloper", lean: 0 },  // 干涉者，lean<0 偏法 / >0 偏英
    scotland:{ role: "interloper", lean: -1 },
    flanders:{ role: "interloper", lean: 0 },
    papacy:  { role: "interloper", lean: 0 },
  },
  // 阶段状态机
  phase: "standoff",        // standoff 对峙 | open_war 鏖战 | truce 疲惫议和 | resolution 定局
  meters: { standoff: 0, open_war: 0, truce: 0 }, // 各阶段计量分
  phaseSinceTurn: 1,
  // 终局
  resolved: false,
  ending: null,             // france_hegemony | england_claim | negotiated_peace | stalemate
}
```

阶段状态机：

| 阶段 | 含义 | 默认漂移（时间诱因） | 激活的局势修正 |
|---|---|---|---|
| `standoff` 对峙 | 王位主张与封建争议，外交博弈为主 | →open_war | 外交操作打折；大规模军事行动诱因分低 |
| `open_war` 鏖战 | 大会战、攻城、占领 | →truce | 军事诱因分最高；战争疲惫累积加速 |
| `truce` 疲惫议和 | 疲惫高、议和便宜、休战期 | →standoff 或 →resolution | 议和谈判操作打折；战争行动受限 |
| `resolution` 定局 | 满足条件可推终局决议 | （终态） | 仅开放终局决议 |

阶段流转规则：每季 `processStruggles` 给「当前阶段的默认下一阶段」+1 漂移分（时间诱因）；诱因事件给对应阶段额外加分；某阶段计量分过阈值（如 ≥10）即翻阶段，清零计量。终局阶段需满足「三段使命」门槛进入（见 §8）。

## 5. 参与度判定 `involvement(world, polity, struggle)`

- **当事国 principal**：首都在 `regionTiles` 内、且是 `parties` 里的 principal（法兰西、英格兰）。可诱因、可用全部局势操作、可推终局决议。
- **干涉者 interloper**：在 `parties` 里标 interloper（勃艮第/苏格兰/弗兰德斯/教皇国）。可选边（改 `lean`）、加诱因分、有干涉收益/惩罚，但**不能推终局决议**。
- **旁观 bystander**：其余国家。局势界面不显示操作入口。

MVP：判定用静态 `parties` 表 + 首都归属；动态加入（如别国参与）放后续。

## 6. 诱因计量 `catalysts`

诱因 = 把游戏里已发生的事件折算成阶段分。MVP 诱因源（全部读现有引擎产出，不新造事件）：

| 诱因事件 | 来源引擎 | 加分 |
|---|---|---|
| 打赢一场会战 | warfare `resolveBattle` | +open_war |
| 占领核心争议城 | warfare `advanceOccupation` | +open_war |
| 签停战 / 王室联姻 | diplomacy `concludePeace` / 联姻 | +truce |
| 统治者更替 / 财政崩溃（money<0 连续） | history / economy | +truce |
| 时间流逝 | `processStruggles` 每季 | +默认下一阶段 1 |
| 干涉者选边 | 局势操作改 `lean` | +对应方阶段 |

实现：`processStruggles` 每季扫描这些信号 → 累加到 `struggle.meters` → 判定翻阶段 → 把诱因与翻阶段写进季度账本（供「本季总结」展示，对齐 france-campaign 计划 Task 3.2）。

## 7. 阶段限定操作（接行动点 + 现有引擎，不另造战斗）

| 操作 | 可用阶段 | 参与度 | 调用 |
|---|---|---|---|
| 提王位主张 | 对峙 | 当事国 | objectives / diplomacy |
| 决战集结 | 鏖战 | 当事国 | warfare 现有军团/会战 |
| 有利停战谈判 | 议和 | 当事国 | diplomacy `concludePeace` |
| 选边支持 | 任意 | 干涉者 | 改 `lean` + 诱因 |
| 终局决议 | 定局 | 当事国 | history 结算（§8） |

所有操作只做「gate + 失败给中文 toast + 调现有引擎」，不绕过行动点、不新增战斗规则。

## 8. 终局决议（复用三段使命状态）

不另立阈值。终局判定读 france-campaign 计划 Task 1.1 的**三段使命状态**（数据驱动 `CAMPAIGN_STAGES`）：

| 终局 | 进入条件（基于三段使命 + 阶段） | 后果 |
|---|---|---|
| 法兰西霸权 `france_hegemony` | 三段全完成 + 处于定局阶段 | 加莱外英格兰退出大陆；法兰西核心永久产出/合法性加成 |
| 英格兰主张得逞 `england_claim` | 英格兰占据核心法兰西争议地 | 英占核心地块，法兰西核心崩坏 debuff |
| 谈判和平 `negotiated_peace` | 议和阶段达成双方妥协 | 争议地分割，双方解除战争疲惫 |
| 长期僵局 `stalemate` | 到第 12 季（样板局终点）未达上述任一 | 双方背疲惫 debt，局势转入低烈度 |

终局只作为样板局结果，**不影响沙盒继续游玩**（与 france-campaign 计划 Task 5.1 一致）。第 12 季对三段使命状态拍快照触发结算。

## 9. UI：局势界面

- **入口**：军事抽屉「战争」页改成局势作战室（对齐 france-campaign 计划 Task 2.2）；御前会议「国家使命」区展示三段使命进度（Task 1.2）。
- **当事国视图**：阶段 + 阶段计量条、三段使命进度、战争分/疲惫、我方主力军、敌方威胁、阶段限定操作、终局决议预览。
- **干涉者视图**：阶段 + 选边态势 + 选边操作 + 干涉收益/惩罚。
- **旁观**：只在世界纪闻里看到局势阶段变化，无操作入口。
- 组件不遮挡右上资源栏与左下地块详情（对齐 layout 测试）。

## 12. 全局约束（沿用项目规范）

- 每个脚本 IIFE，全局挂 `window.HIFI_*`；引擎层不碰 DOM。
- 新增/改脚本后更新 `index.html` 对应 `?v=N`。
- 所有失败原因必须是中文 toast，不吞错。
- 先写测试，再实现；每个 Task 单独 commit。
- 局势新增逻辑只加测试、不改存量 37 个测试的旧断言。

## 13. 验收标准

| 验收项 | 标准 |
|---|---|
| 局势成立 | 法兰西开局存在「百年战争」局势对象，含阶段与参与国 |
| 阶段流转 | 诱因事件 / 时间漂移能推动阶段在四阶段间流转 |
| 参与度 | 当事国有完整局势面板；干涉者只能选边；旁观无操作入口 |
| 操作受限 | 阶段限定操作只在对应阶段可用，否则中文 toast |
| 压力存在 | 鏖战阶段连续跳季不处理，战争疲惫/军需变差且写进账本 |
| 终局 | 12 季内触发四终局之一，复用三段使命状态判定 |
| 回归 | 被动情势（黑死病/小冰期）行为不变；全部 hifi 测试通过 |
