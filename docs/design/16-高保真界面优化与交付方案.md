# 《帝国的代价》高保真界面优化与交付方案

> 状态：静态原型阶段  
> 边界：本阶段不修改 `prototype/demos/帝国的代价-微信小游戏demo2.html`，不接入游戏逻辑。

## 一、交付目标

把现有机制重新组织成一套适合横屏微信小游戏的地图主界面。玩家不需要先理解全部系统，也能立即看清三件事：

1. 我是谁、国家现在怎么样。
2. 地图上正在发生什么。
3. 这一季度最值得处理什么。

## 二、设计原则

| 原则 | 落地方式 |
|---|---|
| 地图是主角 | 地图始终铺满屏幕，系统界面以悬浮栏、抽屉和情境面板覆盖 |
| 问题驱动决策 | 右侧只展示当前必须关注的局势，不堆叠所有通知 |
| 操作就近出现 | 地块操作在左下，军团操作在右上点数栏下方，系统操作从左侧展开 |
| 信息逐层展开 | 默认只露出结论，点击后再看原因、构成和操作 |
| 保留桌游触感 | 资源、命令、季度结算有清楚的筹码感和状态变化 |
| 不增加新机制 | 原型只表达已确认的国家、经济、外交、军事、发展和地图模式 |

## 三、主界面结构

| 区域 | 内容 | 默认状态 |
|---|---|---|
| 顶部国家栏 | 国旗、统治者、季节、年份、国库、合法性、稳定、行政/外交/军事点 | 常驻 |
| 左侧系统栏 | 国家、经济、外交、军事、发展 | 常驻，点击展开薄抽屉 |
| 右侧局势队列 | 战争、阶层、外交、建设等待处理问题 | 常驻，可逐条处理 |
| 左下情境面板 | 当前地块或城市的核心数据与操作 | 选中地图对象后出现 |
| 底部命令栏 | 征召、建设、外交、改革等高频命令 | 常驻 |
| 右下地图工具 | 政治、地形、经济、军事模式与缩放 | 常驻 |
| 右上季度按钮 | 有待办时显示数量，清空后变为结束季度 | 状态化 |

## 四、系统入口压缩

| 一级入口 | 收纳内容 |
|---|---|
| 国家 | 统治者、政体、阶层、法律、议会、决议 |
| 经济 | POP、产出、市场、贸易、建设、国库 |
| 外交 | 国家关系、条约、使节、附属关系、领导人外交 |
| 军事 | 军团、战争、动员、补给、占领、和平 |
| 发展 | 科技、改革、议程、时代目标 |

左侧只保留五个稳定入口。具体功能通过二级抽屉呈现，避免玩家在地图和九个以上按钮之间反复找路。

## 五、地图模式

| 主模式 | 二级信息 |
|---|---|
| 政治 | 国家、政体、家族、阶层、宗教 |
| 地形 | 地形、气候、河流、控制力 |
| 经济 | POP、产出、贸易品、市场、首都 |
| 军事 | 军事控制、补给、占领、邻近度 |

主界面只展示四个模式按钮。二级模式在点击当前模式后展开，不长期占用地图空间。

## 六、关键交互

| 场景 | 交互 |
|---|---|
| 查看国家系统 | 点击左侧入口，左侧滑出系统抽屉；再次点击或点关闭收起 |
| 查看地块 | 单击地图地块，左下出现地块面板；再次点击地图更新内容 |
| 查看军团 | 点击军团单位，在右上点数栏下方弹出独立军团面板 |
| 处理局势 | 点击右侧条目进入对应界面；完成后条目从队列移除 |
| 结束季度 | 有未处理关键局势时按钮显示“处理待办”；清空后显示“结束季度” |
| 切换地图模式 | 点击右下模式按钮即时切换；当前模式保持明确选中状态 |
| 缩放地图 | 滚轮、双指缩放或右下按钮缩放，缩放中心跟随指针或双指中心 |

## 七、视觉规范

| 项目 | 规则 |
|---|---|
| 主色 | 深墨绿用于框架，旧金用于边界，法国蓝用于玩家政权 |
| 状态色 | 红色表示危机，琥珀色表示警告，绿色表示稳定或完成 |
| 面板 | 低透明深色底，细金属边框，圆角不超过 6px |
| 字体 | 中文系统字体；标题使用宋体气质字体，数据使用等宽数字 |
| 控件 | 横屏触控热区不小于 44px，重要按钮不依赖纯图标表达 |
| 动效 | 抽屉 180–240ms，数字与状态变化 120–180ms，不使用装饰性漂浮动画 |

## 八、响应式规则

| 宽度 | 调整 |
|---|---|
| ≥ 1280px | 完整展示国家栏、局势队列、地块面板和命令栏 |
| 900–1279px | 压缩资源标签；局势队列变窄；命令栏减少说明文字 |
| 横屏手机 | 局势队列默认折叠为计数按钮；系统栏缩窄；面板高度受限并可滚动 |
| 竖屏 | 提示旋转设备，不承载正式游戏操作 |

## 九、分阶段接入

| 阶段 | 工作 | 完成标准 |
|---|---|---|
| 静态原型 | 独立 HTML 验证布局、视觉、响应式和基础点击状态 | 用户确认布局与视觉 |
| 界面外壳 | 在副本中建立 HUD 组件和统一状态，不动机制计算 | 新旧机制数据能映射到新界面 |
| 地图接入 | 接入选地、缩放、地图模式和迷雾 | 地图交互不回退 |
| 系统接入 | 依次接入国家、经济、外交、军事、发展抽屉 | 每次只替换一个入口并验证 |
| 情境面板 | 接入地块、国家和军团独立面板 | 面板之间不复用错误状态 |
| 季度闭环 | 接入局势队列、待办计数和季度结算 | 完整跑通一季度 |
| 清理验收 | 删除旧界面冗余代码，进行横屏设备回归测试 | 无重复入口、无阻断问题 |

## 十、原型验收清单

- [ ] 第一眼能识别玩家国家、统治者和当前季度。
- [ ] 地图可视面积不低于屏幕的 65%。
- [ ] 五个系统入口含义清楚，没有重复入口。
- [ ] 局势队列能说明“为什么现在需要行动”。
- [ ] 地块面板、军团面板和系统抽屉位置互不冲突。
- [ ] 地图模式和缩放操作集中在右下角。
- [ ] 横屏手机上没有文字重叠或不可点击控件。
- [ ] `demo2` 在用户确认前保持原样。
