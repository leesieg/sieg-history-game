# 20 · 经济 / 军事 / 外交 / 发展机制深化设计

> 适用范围：`prototype/hifi/` 主线的「经济 / 军事 / 外交 / 发展」四个面板。
> 范式承自 [`19-国家机制深化设计.md`](19-国家机制深化设计.md)：每条机制都要回答**受什么流影响 / 影响什么流 / 现状如何展示**，空转机制（拧了不影响任何流）优先接进核心闭环。核心循环见 `CLAUDE.md` 的「核心循环」一节。

## 0. 共同范式

世界是一台每季自转的机器：**基底（地块·人口·资源）→ 流（产出 / 贸易 / 人口 / 科技 / 信仰）→ 压力层 → 转折层 → 回基底**。下面逐模块审计「哪些阀门已接到流上、哪些在空转」，再给空转项的接入设计。

审计图例：✅ 已接入闭环　🟡 部分接入 / 有缺口　🔴 空转（只写不读 / 无下游）

## 1. 经济（economy.js + trade.js + rules.js）

### 1.1 现状审计

| 机制 | 状态 | 说明 |
|---|---|---|
| 地块产出流 `tileOutput`→`settleCountry` | ✅ | 人口×控制度×(1−破坏)×(1−占领)，建筑/科技乘数，是产出流的源头 |
| 建筑 buildings | ✅ | farm/market/port/fort/workshop 真实乘数 |
| 贸易流 `route.flow`→`lastIncome` | ✅ | 路线 value×(1+boost)×成本衰减；节点国按关税分成入账 + 资本池 |
| 关税 tariff | ✅ | 抬高本国分成，但同时抬高路线成本（压低流）与财政/贸易压力——真实取舍 |
| 商路投资 `investRoute` | ✅ | boost 被 processTrade 消费（doc 18 已接） |
| 敕令 edicts | ✅ | 真实资源增减 |
| 议程 agendas | ✅ | 达标真实奖励并清空 |
| 科技采纳 `adoptTechnology` | ✅ | 见第 4 节 |
| **贸易政策 tradePolicy** | 🔴 | 仅 `open` 有效果（+12% 钱 + 资本）；`closed` 与 `normal` **完全等价、无任何差别** |
| **资本池 capital** | 🔴 | 从开放贸易 / 贸易收入 ×.15 累积，**全局无任何消费点**——只涨不用的死池 |
| **银库 pools.silver** | 🔴 | newWorld 路线注入，**无消费点** |
| **压力层 pressures** | 🟡 | trade/military/fiscal/exploration/faith/ideas 每季算出，但**只有 exploration≥35 喂探索点**，其余只展示、不驱动任何流或转折 |

### 1.2 受什么流影响 / 影响什么流

- **被影响**：产出流来自基底（地块人口 / 控制度 / 战争破坏 / 占领）；贸易流来自地块节点 + 路线成本（受关税、devastation、占领、君堡陷落抬升）。
- **影响**：产出流 → 国库 / 粮仓 / 军需；贸易流 → 国库 + 资本池 + 压力层（trade/fiscal）。

### 1.3 接入闭环的设计

1. **贸易政策给三档真实差异**（消灭 🔴）：
   - `closed` 封闭：产出流金钱 ×0.95、贸易流分成 ×0.6，但**贸易/外部依赖压力 −**、对外部断供的抗性 +（战时不被掐贸易）。
   - `normal` 常规：基准。
   - `open` 开放：保留现有 +12% 钱 + 资本，但**贸易压力 +**、战时易被封锁。
2. **资本池 capital 给消费出口**（消灭 🔴）：作为「一次性大投资」的货币——建造高级建筑（workshop/港口升级）、赎买地块、资助商社扩张路线。让资本池成为「贸易流的蓄水池 → 转为基底改造」的桥。
3. **银库 silver 接物价**：silver 流入 → 抬 `priceIndex`（见第 4 节物价），把「新大陆白银 → 价格革命」这条历史因果接通。
4. **压力层真正驱动转折**：fiscal 压力高 → 阶层（平民/商人）满意 −；military 压力高 → 战争疲惫加速 + 合法性 −；faith 压力高 → 宗教情势/改革宗诉求；trade 压力高 → 触发商业扩张事件。让 `pressures` 从「仪表盘」变成「压力层→转折层」的真实输入。

### 1.4 展示 / 百科

- 资本池、银库、各压力条加 codex tooltip（受什么影响 / 影响什么）。
- 贸易政策三档 detail 标出各自取舍（而非现在的「稳定 / 均衡 / 资本增长」空话）。

## 2. 军事（warfare.js）

### 2.1 现状审计

| 机制 | 状态 | 说明 |
|---|---|---|
| 动员 `mobilizeArmy` | ✅ | **人口流→军队**：征召扣地块 POP；炮兵扣军需 |
| 补员 `reinforceArmy` | ✅ | 军需流→兵员（×20/点） |
| 佣兵 hire/renew/release/contracts | ✅ | 金钱流维护 + 忠诚 + 合同到期，欠饷掉忠诚 |
| 训练 trainArmy | ✅ | 军事点 + 军需 → 经验 / 组织度 |
| 将领 generals | ✅ | 统治者三维 → 指挥 / 攻城 |
| 行军 movement / 地形 | ✅ | 地形成本、组织度损耗、BFS 路径 |
| 战斗 resolveBattle | ✅ | 兵种 × 地形 × 经验 × 士气 × 组织 × 补给；伤亡 → 地块破坏 + POP 流失（**回基底**） |
| 占领 advanceOccupation | ✅ | 占领 → 战争分数 + 战争疲惫 + 占领地产出流归零 |
| 议和 concludePeace | ✅ | 领土转移 + 停战 + 触发君堡因果链 |
| **战争疲惫 warExhaustion** | 🟡 | 占领时累积，喂 military 压力 + 顾问预警，但**没有硬性惩罚**（不削产出 / 不掉合法性 / 不限募兵） |
| **兵员/人口恢复** | 🟡 | 募兵直接扣 POP、复员还 POP，但 POP 本身**无自然恢复流**，长期战争把地块抽干后不可逆 |

### 2.2 受什么流影响 / 影响什么流

- **被影响**：军队规模受人口流（募兵）/ 军需流（补员）/ 金钱流（佣兵）约束；战斗力受补给（己方领土）影响。
- **影响**：战斗 → 地块破坏 + POP 流失（直接改基底，压低未来产出流）；占领 → 产出流归零 + 战争疲惫；议和领土转移 → 永久改基底归属。

### 2.3 接入闭环的设计

1. **战争疲惫长出牙齿**（🟡→✅）：warExhaustion 每季按值
   - 拉低合法性、压低全阶层满意度（接 doc 19 的阶层惩罚）、
   - 超阈值时**封顶可募兵力**（人口流被战争意志限制），
   - 和平后逐季消退。
   让「打太久」有真实代价，而不只是一句预警。
2. **人口自然恢复流**：非战乱地块每季 POP 缓增（受粮食 / 控制度 / 破坏度调节），把「人口」做成真正的流而非一次性资源；战争破坏 devastation 也逐季自愈。这样长期战争与休养生息形成张力。
3. **动员制度联动（接 doc 19 法律）**：`mobilization` 法律（有限/征召/常备）调节募兵 POP 成本与上限、常备兵质量与军需维护——军事与政治两个阀门共用「人口流→军队」这条管道。

### 2.4 展示 / 百科

- 战争疲惫、补给、组织度、士气加 codex tooltip。
- 军团行 detail 标出「补给 / 组织 / 士气」三态对战斗力的影响。

## 3. 外交（diplomacy.js）

### 3.1 现状审计

| 机制 | 状态 | 说明 |
|---|---|---|
| 关系多维 trust/threat/favor/conflict/strategicInterest | ✅ | 决定态度与提案评分 |
| 领导人关系 friendship/respect/fear/grudge/kinship | ✅ | 喂提案评分 |
| 使节 missions（improve→trust） | ✅ | 真实改善关系 |
| 外交容量 capacity | ✅ | 统治者外交 + 政治改革；超容量掉外交点 |
| 朝贡 / 附庸 / 傀儡 subjects | ✅ | **贡赋流**：附属国→宗主金钱；忠诚 / 自主权 / 收紧放宽 |
| 贸易协定 trade treaty | 🟡 | 每季 +10 金钱，但是**固定平摊、与真实贸易流无关** |
| **同盟 / 联姻 / 互不侵犯 / 通行** | 🔴 | 只改 trust + 占容量，**没有结构性后果**：同盟不会被拉入战争、联姻不影响继承、互不侵犯不阻止宣战 |
| **威胁 / 战略利益 / 领土矛盾** | 🟡 | 初始化后基本静态，`territorialConflict` 恒为 0，不随接壤 / 战争 / 共享商路演化 |

### 3.2 受什么流影响 / 影响什么流

- **被影响**：实力评估 `countryStrength` 来自基底（人口 + 领土 + 军需）；外交容量来自统治者三维 + 政治改革。
- **影响**：贡赋流改双方金钱；贸易协定改金钱流；关系决定战争 / 提案可行性。

### 3.3 接入闭环的设计

1. **关系随流演化**（🟡→✅）：
   - 接壤地块数 → `territorialConflict`；交战历史 → grudge / threat；
   - **共享同一条贸易路线 → strategicInterest +**（贸易流反哺外交）；
   - 实力差距扩大 → 弱方 threat +。
   让外交态度成为世界状态的函数，而非初始常量。
2. **条约长出结构性牙齿**（消灭 🔴）：
   - 防御同盟：被宣战时同盟国按意愿自动加入 `war.defenders`；
   - 王室联姻：影响继承候选 / 提供「联合」声索（接 politics 的继承）；
   - 互不侵犯 / 停战：已在 warfare `underTruce` 阻止宣战（联通），UI 标注；
   - 贸易协定：改为**按双方实际贸易流的比例**给收益，而非固定 +10。
3. **AI 主动外交**（strategy.js）：AI 依据态度 / 实力主动发起改善 / 结盟 / 宣战，让外交场是活的。

### 3.4 展示 / 百科

- 信任 / 威胁 / 战略利益 / 领土矛盾加 codex tooltip（说明各自的演化来源）。
- 条约行 detail 标出「结构性后果」（如同盟「被攻击时共同防御」）。

## 4. 发展（history.js + economy.adoptTechnology + rules.technologies）

### 4.1 现状审计

| 机制 | 状态 | 说明 |
|---|---|---|
| 科技扩散流 `spreadTechnology` | ✅ | 邻国已知数 + 印刷术 → 传播度；时代进度 |
| 科技采纳 adoptTechnology | ✅ | 思想点 + 年代 + 传播度门槛；改各流（产出 / 军需 / 解锁路线 / 炮兵） |
| 时代使命 missions | ✅ | 真实资源奖励 |
| 情势 situations | ✅ | 黑死病扣 POP（回基底）、小冰期；事件选项真实增减 |
| 历史因果链 君堡陷落 | ✅ | 探索压力 + 物价上行 + 转折 |
| 纪元 eras | 🟡 | 按年份推进、推世界事件，但**进入新纪元几乎不改可用机制 / 规则**（决议靠 reformation flag、科技靠 year，不靠 eraIndex） |
| **探索点 / 里程碑 exploration** | 🔴 | 探索压力 → 探索点 → atlantic 里程碑，但里程碑**无任何奖励 / 解锁** |
| **物价指数 priceIndex** | 🔴 | 君堡因果链 +.04，但**全局无消费点**——不影响产出 / 贸易 / 满意 |
| **压力层 faith / ideas** | 🟡 | 算出但只展示；faith 不驱动宗教不满 / 改革宗诉求，ideas 不加速科技 |

### 4.2 受什么流影响 / 影响什么流

- **被影响**：科技扩散流受邻国采纳数 + 印刷术驱动；探索流受贸易压力 + 发现冲动驱动。
- **影响**：科技 → 产出 / 军需 / 贸易解锁；情势 → 基底（POP）；因果链 → 压力 + 物价 + 转折。

### 4.3 接入闭环的设计

1. **探索里程碑给真实回报**（消灭 🔴）：atlantic / 新航线里程碑 → 解锁跨洋贸易路线、注入殖民收入流、推时代进度——把探索流接回贸易流与转折层。
2. **物价指数接产出与贸易**（消灭 🔴）：priceIndex 调节金钱产出流与贸易分成（价格革命：白银流入 → 物价上行 → 名义收入升但实际购买力降 / 阶层不满）。把银库（第 1 节）→ priceIndex → 产出/满意 这条历史因果跑通。
3. **faith / ideas 压力驱动转折**：faith 高 → 异端不满 / 触发改革宗诉求（接 doc 19 宗教法律）；ideas 高 → 加速科技扩散流（spreadTechnology 速率 +）。
4. **纪元真正改规则**：进入新纪元解锁 / 关闭决议与法律选项（如绝对主义纪元开放绝对权力路线、革命纪元施压共和），让转折层真正反作用到机制可用性。

### 4.4 展示 / 百科

- 探索点 / 里程碑 / 物价指数 / 时代进度加 codex tooltip。
- 时代使命行标出奖励；纪元转折说明「本纪元解锁 / 关闭了什么」。

## 5. 实现优先级（跨模块）

> **实现状态（持续更新）**：阶段 1–6 全部落地并测试通过。阶段 3 的余项也已补齐：王室联姻→王朝纽带 kinship（持续抬升信任 + 提升从属提案接受度，作为「和平继承式整合」的近似；完整的人格联合/王位继承仍可作未来扩展）、共享商路→战略利益、贸易协定按真实贸易流计酬、AI 主动外交（缔约/派使节）均已实现。

按「消灭空转 🔴 优先、低风险先行」排序，每阶段配测试、改脚本后递增 `index.html` 的 `?v=N`：

| 阶 | 内容 | 模块 | 风险 |
|---|---|---|---|
| 1 | 贸易政策三档差异 + 资本池消费出口 | 经济 | 低 |
| 2 | 战争疲惫硬惩罚 + 人口自然恢复流 | 军事 | 中 |
| 3 | 条约结构性后果（同盟参战 / 联姻继承）+ 关系随流演化 | 外交 | 中 |
| 4 | 探索里程碑回报 + 物价指数接产出/贸易（白银→物价→满意） | 发展 | 中 |
| 5 | 压力层全面驱动转折（fiscal/military/faith/ideas → 满意/疲惫/宗教/科技） | 经济+发展 | 中高 |
| 6 | 纪元改规则（解锁/关闭决议与法律） | 发展+国家 | 中 |

> 与 doc 19 的衔接：法律 / 改革的流效果（doc 19 的 B–E）与本文第 1–2 节的产出流 / 人口流改造**共用同一批管道**（tileOutput / settleCountry / mobilizeArmy），建议合并实现以免重复改动。
