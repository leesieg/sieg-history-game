# 34-当前版本优化方案

> 依据：`33-当前游戏版本整体评测报告.md`（评测对象 `prototype/hifi`）
> 配套：`30-整体评测方案.md`（评测分层与 `tools/sim` 规格）
> 方案日期：2026-06-25

## 0. 总纲

33 号报告的判断是：**当前版本已经是"可评测的策略框架"，但还不是"稳定好玩的策略游戏"**。
核心矛盾不是功能少，而是**系统压力不稳定 + 部分机制空转**：裁断长期为 0、从属 0 次触发、建设/科技在 40 季样本里几乎不动。

本方案把 33 号§6 的优先清单翻译成**可落地的代码动作**，并严格遵守 33 号§7 的"不做"边界：

- 不直接大改资源产出（没有正式 L2 基线之前调数值只会越调越乱）。
- 不堆新系统（先把从属/裁断/建设这些**已存在但空转**的机制接进闭环）。
- 不把弱国直接补成强国（补"可操作选择"，不补结果）。
- 不把所有待办都变成阻塞（阻塞只留给真正需要玩家二选一的事）。

执行顺序按"成本低→收益确定"排：**P0 UI 明确 bug → P1 机制激活（从属 / 裁断）→ P2 小国操作空间 → P3 落地 `tools/sim`**。每个 Phase 收尾都跑 `for f in tests/*.test.cjs; do node "$f"; done`。

## P0 · UI 明确缺陷（成本低、错误明确）

### P0-1 地形横幅 404

**根因（已核对，比报告更精确）**：
- `prototype/hifi/styles/components.css:804` 用 `url("../../assets/terrain-banners/plains.png")`。CSS 的 `url()` 相对**样式表自身**解析：从 `prototype/hifi/styles/` 出发 `../../assets/` = `prototype/assets/`，**该目录不存在**，真实目录是仓库根 `assets/terrain-banners/`。→ **这是唯一真的 404**。
- `prototype/hifi/scripts/ui/map.js:472` 的内联 `background-image` 相对**文档（index.html）**解析：从 `prototype/hifi/` 出发 `../../assets/` = 仓库根 `assets/` → **实际正确**。

**动作**：
- 仅改 CSS：`components.css:804` 的 `../../assets/` → `../../../assets/`。
- **不要动 map.js**——它的路径是对的，"修"它反而会引入 404。
- 改完递增 `index.html` 里 `components.css` 的 `?v=N`。

**验收**：本地 8765 打开地图详情，9 种地形横幅（plains/forest/hills/mountains/coast/sea/desert/steppe/wetland）都能加载，Network 无 404。

### P0-2 弹窗与系统抽屉同时占屏

**根因（已核对）**：`main.js:36 openModal()` 只遍历 `.modal-layer` 关闭其它弹窗，**不关系统抽屉**。抽屉由 `openSystem()`(main.js:510) 单独控制，二者无互斥。打开局势/国家弹窗时外交抽屉仍 `open`。

**动作**：在 `openModal(id)` 末尾追加"关抽屉"：
```js
// 打开任意主界面弹窗时，统一收起系统抽屉与其 active 态
drawer.classList.remove("open");
drawer.setAttribute("aria-hidden", "true");
document.getElementById("game").classList.remove("system-open");
document.querySelectorAll(".system-button").forEach(b => b.classList.remove("active"));
```
（`drawer` 在 main.js 模块作用域已可用；若不可用则 `document.getElementById("systemDrawer")`，按实际 id 取。）改完递增 `main.js` 的 `?v=N`。

**验收**：打开外交抽屉 → 点百年战争待办打开局势弹窗 → 屏幕只剩 `struggleModal`，抽屉收起、系统按钮无 active。`hifi-ui-smoke` / `hifi-layout` 仍绿。

## P1 · 机制激活：让"已设计但空转"的机制真正触发

### P1-1 从属机制 0 次触发

**根因（已核对）**：AI 其实**会**尝试索附属（`strategy.js:214 subjectTarget` → `proposeSubject`），但 `diplomacy.js:225 evaluateProposal` 的 `tributary` 几乎永远 `accepted=false`：

- `subjectTypes.tributary`：`base 34 / threshold 58`（diplomacy.js:13）。要达成需净 +24。
- Phase E 平衡补正（diplomacy.js:247-252）把"实力差距"封顶 +30，又对 `targetPower>=40` 叠"独立意志"惩罚、对从属叠"主权损失" -10。
- 叠加结果：**强国想压服稍有体量的邻国几乎不可能，能压服的极弱邻又凑不够 58**——形成"谁都签不成"的死区。这是过度修正（当初是为了消除"碾压即附庸"，矫枉过正）。

**动作（三步，先通后调）**：
1. **打开可行窗口**：把 `tributary` 的 `threshold` 由 58 降到 ~48（或 `base` 34→40），让"强国 vs 明显更弱的接壤邻国"能成立；`vassal` 同步小幅放宽。**只动 tributary/vassal 这两个常数，不动评分项结构**，便于 P3 用 `tools/sim` 回归。
2. **玩家侧引导**（drawers.js 从属页 `subject:` 行已存在，约 drawers.js:530）：对每个可选目标显示 `evaluateProposal` 的 `score / threshold` 与是否 `accepted`，把"哪些邻国可从属"直接标出来（复用 33 号已验证的"可从属目标"概念）。
3. **宗主收益预览**：在从属页展示该 `subjectType` 的 `tribute`/capacity 与"建立后每季贡赋"，让玩家知道为什么要做。

**验收**：用临时 Node 统计脚本（同 33 号§3 口径，34 国 × 40 季）重跑，**从属关系中位数 > 0**；玩家从属页能看到至少一个标"可从属"的邻国并给出分数。

### P1-2 阻塞裁断长期为 0

**根因（已核对）**：`history.js:349 issues()` 里 `blocking:true` 只有三个来源——`world.pendingElection`、`world.playerEvents`、以及"爆发且 eventGenerated"的 situation（history.js:351-360）。40 季内这三者都几乎不触发，于是"必须裁断"恒为 0，御前会议退化成纯提示板。

**动作**：新增**少量、真正二选一**的裁断事件，挂在已有的"压力层"流上（符合闭环设计，不是新系统）：
- **财政危机**：玩家 `money` 触 0 / 连续 N 季赤字 → 生成一条 `playerEvent`（如"举债 vs 增税 vs 削减开支"），三选项各自反作用到 money/合法性/阶层满意。
- **阶层冲突**：某阶层满意度跌破阈值 → `playerEvent`（"安抚 vs 镇压"），接合法性/产出流。
- **情势升级**：现有 situation 进入"爆发"时，确保 `eventGenerated=true` 真能产出一条 blocking 事件（核对该字段当前是否真被置位——若从不置位，这就是 situation 路径恒不 blocking 的二级原因）。

实现落点：在 `processHistory`（或对应每季处理）里检测上述条件 → `world.playerEvents.push({...})`；裁断 UI 与 `blockingIssues`/`seasonControl` 拦截（main.js:556 已有 `仍有 N 项裁断需要处理` 逻辑）天然复用。

**约束**：每季阻塞裁断**总量克制**（目标中位数 1，峰值 ≤2），严守 33 号§7"不把所有待办都变阻塞"。

**验收**：34 国 × 40 季重跑，**阻塞裁断中位数 ≥1 且 ≤2**；裁断弹出时 `seasonControl` 正确拦截结束回合。

## P2 · 小国操作空间（补"可选择"，不补"结果"）

**现状（33 号§3.2/3.3）**：8/34 样本合法性归 0、5 个最弱国 money=0/合法性=0。问题不是"难"，是"无操作空间"。

**方向（不改产出曲线，只加可主动触发的恢复手段）**：
- 低合法性时解锁**廉价合法性动作**（如"召开等级会议/加冕仪式"——花行政点换合法性回升），给玩家"我还能做点什么"的出口。
- 破产时与 P1-2 的"财政危机裁断"联动：举债选项让小国不至于动作归零。
- **合法性下限保护**：核对是否存在地板值；若直接归 0 锁死所有自由动作，给一个极低但非 0 的可恢复地板（保留操作空间，不等于补强度）。

**注意**：具体数值（阈值、回升量、地板）**留到 P3 `tools/sim` 落地后再调**——本 Phase 只搭"可操作手段"的骨架，不拍数值。

**验收**：最弱样本（纳瓦拉/挪威/苏格兰等）在 40 季内**始终有 ≥1 个可执行的自由动作**（合法性恢复或举债），合法性归 0 样本占比从 8/34 下降。

## P3 · 落地 `tools/sim`（正式 L2 基线，数值调平的前提）

按 30 号§4 建（**不进 `tests/`**，产出数据而非断言）：

```
tools/sim/
  run.cjs       # 复用 hifi-longrun 的 vm.runInNewContext 装载方式；N 局 × M 季 × 策略画像；导出原始 JSON
  metrics.cjs   # 算 30 号§4.2 三类指标分布（稳定性/资源/机制活性）
  report.cjs    # 生成对齐 29/33 号表格的 markdown
  baselines/    # 各版本指标快照，供 diff
```

- 策略画像沿用 passive/balanced/military/economy/diplomacy。
- 一律走引擎层纯 Node（30 号踩过坑：浏览器 100 局超 5 分钟工具上限）。
- **把 33 号§3 用过的临时统计脚本固化进 `run.cjs`/`metrics.cjs`**，避免每次评测重写。

**验收**：`node tools/sim/run.cjs` 100 局 × 40 季分钟级完成产出 JSON；`report.cjs` 一键生成报告；P1/P2 改动**前后各跑一次 diff**，确认从属中位数 >0、裁断中位数 1、且资源/完成率指标未显著恶化。
