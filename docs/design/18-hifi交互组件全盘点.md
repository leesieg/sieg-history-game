# 18 · hifi 主线交互组件全盘点

> 对象：`prototype/hifi/`（模块化原生 JS 主线）。
> 内容：把 demo 的**每一个可交互组件**(按钮、遮罩、输入框、列表项、地图点击区)逐一列出——触发交互与影响——并标注合理性。
> 配套：17 号《机制现状说明与待修问题》、`prototype/diagrams/hifi主线机制与问题框架图.html`。
> 状态：本文为「交互组件层」盘点，已反映两轮修复(死按钮清除 + 弱交互整改)后的现状。

通用机制：系统抽屉内 `data-*` 动作按钮经 `main.js` 的 `runAction` 执行 → 成功则 `store.update` 触发全局重渲染(HUD + 地图 + 当前抽屉)；失败(引擎 `throw`)弹 toast 并保持状态不变。

## 二、全局交互区

### 顶部指挥栏
| 组件 | 交互 → 影响 |
|---|---|
| 统治者头像 `rulerPlaque` | 完成「查看国家」教程 + 打开国家详情弹窗 |
| 待办 N › `topPending` | 打开御前会议 |
| 7 个资源 token | 只读显示，非按钮 |

### 左侧系统轨(5 键 `data-system`)
| 组件 | 交互 → 影响 |
|---|---|
| 国家 / 经济 / 外交 / 军事 / 发展 | 打开/关闭对应抽屉;再次点同键=关闭;打开时关军团抽屉、让出右侧问题区 |

### 季度控制
| 组件 | 交互 → 影响 |
|---|---|
| 结束季度 / 处理裁断 N `seasonControl` | 有阻塞裁断则 toast 拒绝;否则 `advanceQuarter` 推进一季(战争→产出→贸易→外交→继承→历史→AI) |

### 右侧问题面板
| 组件 | 交互 → 影响 |
|---|---|
| 问题条目 ×N `data-history-issue` | 事件类→历史事件弹窗;其余→御前会议 |

### 地图舞台(SVG 点击区，非 `<button>`)
| 组件 | 交互 → 影响 |
|---|---|
| 地块 `data-map-tile` | 选中地块、刷新省份面板;军团规划态下→设该军团路线 |
| 军团标记 ♞ `data-army-marker` | 打开该军团抽屉 |
| 缩略图 `miniMapSvg` | 视图居中到点击处并放大 |
| 主区滚轮 / 拖拽 | 缩放 / 平移(位移<5px 视为点击) |
| 10 透镜 `data-mode` | 切换地图着色 + 图例(政治/地形/人口/产出/贸易/宗教/家族/政体/阶层/军事) |
| 图 `legendToggle` / ◎ `zoomReset` / − `zoomOut` / + `zoomIn` | 图例开合 / 复位 / 缩小 0.3 / 放大 0.3 |

## 三、系统抽屉内动作按钮

### 国家抽屉(politics)
| 按钮 | 成本 → 效果 |
|---|---|
| 改革槽 ×6 `data-reform` | 升 1 级(满 5 报错);扣对应资源(行政→10钱、军事→10军需、政治→4合法性、海事→10钱) |
| 法律 ×4 `data-law` | 循环切下一档;扣 1 行政点 |
| 议会 ×3 `data-assembly`(让渡特权 / 收买议会 / 强硬施压) | 扣 1 行政点;特权 +15 支持(阶层权力+2)、收买 +10(−12 金钱)、强硬 +0 |
| 国家决议 ×6 `data-decision` | 满足前置才执行转折，否则 toast 说明原因 |
| 整合〔选中地块〕`data-integrate` | 扣 20 钱 + 1 行政点;控制度 +20(满 100 报错) |

### 经济抽屉(economy + trade)
| 按钮 | 成本 → 效果 |
|---|---|
| 贸易政策 ×3 `data-trade-policy` | 设策略(开放=结算额外金钱+资本)+ 完成教程 |
| 关税 ×3 `data-tariff`(0/10/25%) | 设关税(影响贸易收入与财政压力)+ 教程 |
| 贸易路线 ×10 `data-trade-route` | **投资该商路**:扣 15 钱 + 1 行政点，需拥有该路节点;路线 `boost`+15%(上限 60%)→ 结算时提升流量与节点收入;并切到贸易透镜 |
| 敕令 ×3 `data-edict` | 即时扣成本换即时资源 |
| 议程 ×3 `data-agenda` | 设目标，结算达阈值给合法性奖励 |
| 建筑 ×5 `data-building` | 在选中己方地块建造，扣金钱 + 1 行政点 |

### 发展抽屉(history)
| 按钮 | 效果 |
|---|---|
| 科技 ×9 `data-technology` | 需达年代 + 传播度≥25% + 思想点足;采纳后推进时代进度 |

### 军事抽屉(warfare)
| 按钮 | 效果 |
|---|---|
| 动员步兵 / 动员骑兵 `data-mobilize` | 选中己方地块征召(扣 POP + 1 军事点) |
| 铸造炮队 `data-mobilize=artillery`(掌握火炮科技后出现) | 扣 1 军事点 + 30 军需 |
| 雇佣自由佣兵团 `data-hire-mercenary` | 扣 40 钱，生成带契约期/忠诚的佣兵团 |
| 军团条目 ×N `data-army-open` | 打开该军团抽屉 |
| 战争条目 ×N `data-peace-war` | 索取目标领土(战争分≥25)或提议停战(立 20 季停战) |

### 外交抽屉(diplomacy)
| 按钮 | 效果 |
|---|---|
| 外交对象 ×N `data-diplomatic-target` | 切换当前外交目标 |
| 外交行动 ×12 `data-diplomatic-action` | 使节 / 会晤·赠礼·威慑 / 贸易·通行·联姻·互不侵犯·同盟条约 / 朝贡·附庸·傀儡;条约附属走打分，不达阈值 toast 拒绝 |
| 宣战 `data-diplomatic-action=war:declare` | 对当前目标宣战(自动以其首都为战争目标);**已交战/停战期时降级为不可点提示行** |
| 收紧 / 放宽 `data-subject-control`(己方附属) | 扣 1 外交点;调自主/忠诚/贡赋 |

## 四、各弹窗 × 每个可点击组件

### 系统抽屉 `systemDrawer`
- `×` `drawerClose` → 关闭、取消系统轨高亮
- 抽屉体内当前系统的全部动作按钮(见上节)
- 非点击:各 `drawer-row` 数值行(资源/压力/阶层/使命/导师指引)

### 军团抽屉 `armyDrawer`
- `×` `armyDrawerClose` → 关闭
- 规划路线 `data-army-plan` → 进规划态并关抽屉
- 原地防守 / 继续行军 `data-army-order` ×2(无路线行军给提示)
- 拆分 / 补充兵员 / 训练 / 复员征召兵 `data-army-manage` ×4
- 统治者领军↔撤下统治者(国家军团)/ 续约两年·结束契约(佣兵团)
- 合并 ×N `data-army-merge`(同国同地同类军团)
- 非点击:所属/位置/士气组织补给/将领/契约/编制

### 省份面板 `provincePanel`
- `×` `provinceClose` → 隐藏面板(不改状态)
- 建设 / 征兵 / 整合 / 派驻 ×4 → 打开对应抽屉**并滚动定位+高亮**对应动作
- 非点击:气候/人口/产出/控制度(反映占领)/发展度/文化/宗教

### 国家详情弹窗 `countryModal`
- `×` `data-close-dialog` → 关闭
- 点遮罩空白处 → 关闭
- 切换国家 `openCountrySelect` → 关本弹窗、开选国弹窗
- 非点击:徽记/标题/`countryModalBody`(政体/合法性/能力/资源/阶层/决策回响)

### 选择国家弹窗 `countrySelectModal`
- `×` / 点遮罩 → 关闭
- 搜索框 `countrySearch`(`input`)→ 实时按国名/政体过滤
- 国家卡 ×N `country-choice`(`data-polity`)→ 选中并刷新右侧预览
- 确认国家 `confirmCountryChoice` → `setPlayerCountry` 切换扮演国 + 关闭
- 非点击:标题、预览区(简介 + 前 4 项数据)

### 领导人选举弹窗 `leaderElectionModal`(强制裁断)
- 候选人卡 ×3 `data-candidate` → `completeElection` 选定、关闭、解阻塞
- 无 × 无遮罩关闭 —— 必须选一位
- 非点击:标题、选举原因

### 御前会议弹窗 `councilModal`
- `×` `data-close-dialog` → 关闭
- 确认时代转折 `data-ack-transition`(有转折时出现)→ 清除待确认转折并关闭
- 垂帘听政 4 季 `data-run-regency` → 连续自动推进至多 4 季(遇阻塞停)并关闭
- 无遮罩关闭;预警/建议/局势为展示行

### 历史事件弹窗 `historyEventModal`(强制裁断)
- 选项卡 ×N `data-history-choice` → `resolvePlayerEvent` 应用资源效果、关闭、解阻塞
- 无 × 无遮罩关闭 —— 必须选一项
- 非点击:事件标题

## 五、合理性汇总表

| 区域 | 组件 | 合理性 | 说明 |
|---|---|---|---|
| 顶栏/系统轨/季度/问题面板/地图 | 全部 | ✅ | 直接生效 |
| 国家抽屉 | 改革/法律/决议/整合 | ✅ | |
| 国家抽屉 | **议会 ×3** | ✅(已修) | 原仅写死 tax:privilege，现暴露特权/收买/强硬三种让步 |
| 经济抽屉 | 政策/关税/敕令/议程/建筑 | ✅ | |
| 经济抽屉 | **贸易路线** | ✅(已修) | 原 `selectedRoute` 只写不读(🔴 死交互);现为「投资商路」，`boost` 经 `processTrade` 提升流量与收入 |
| 发展/军事/外交抽屉 | 大部分 | ✅ | |
| 外交抽屉 | **宣战(已交战态)** | ✅(已修) | 原已交战仍可点、仅 toast 拦截;现已交战/停战期降级为不可点提示行 |
| 命令坞 | **6 键** | ✅(已修) | 原纯跳转绕路(🟡);现打开抽屉后**滚动定位 + 高亮**对应动作 |
| 省份面板 | **快捷 4 键** | ✅(已修) | 同上，定位到建筑/动员/整合/军团 |
| 军团/各弹窗 | 全部 | ✅ | 强制选择型弹窗无关闭键为有意设计 |

> 截至本盘点，**无残留 🔴/🟡 交互**(空按钮、死写入、纯绕路均已整改)。仅余 17 号文档标注的「待设计确认」机制项(AI 主动性、附属平衡、因果链扩展、发展度/POP 展示口径)，属设计决策而非交互缺陷。
