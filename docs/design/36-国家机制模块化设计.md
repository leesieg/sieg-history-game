# 36-国家系统重构设计（模块化政体）

> 承接 `19`（国家流模型总纲）、`35`（操作-成本-影响全景图）后的设计讨论结论。

# 第一部分 · 设计总述（国家模型）

> 本部分是设计共识本身——重构后「国家」应该是什么样的机器。后面第二部分起才是落地蓝图。

## A. 为什么要重构：现状国家系统的四个病

| 现状 | 病在哪 |
|---|---|
| **政体是死枚举** `monarchy/republic/theocracy/empire/tribal/merchant_republic` | 类型即标签，组合空间为零；想从君主变共和只能走一条写死的 decision；法国君主制和英国君主制在数据上**完全一样**，历史差异无处安放 |
| **改革是抽象的 0-5 数值槽** | 拧「财政改革」只给 +3%/级，没有真选择——是项目最忌的**空转机制**；六个槽（行政/财政/军事/宗教/政治/海事）名实不副 |
| **王权是手动滑块** | `centralPower` 只靠玩家点法律/决议推动，**完全不读外部压力**；战争、邻国威胁算出来了却不影响它，王权与世界脱节 |
| **阶层与宗教被硬绑在政体里** | 阶层集随政体枚举固定；宗教（宽容/正统/改革宗）被塞进政府结构，但它本质是信仰流的事，不是政体结构 |

一句话：**现在的「国家」是一张静态卡片，不是一台会响应世界的机器。**

## B. 核心思想：政体是「涌现」的，不是「选定」的

> **政体 = 五条制度模块的组合。** 不存「政体类型」这个字段，而是存五个模块的当前取值，**原型名、国家名、阶层集、一组产出阀，全部从组合里派生出来。**

五条制度轴（纯结构，宗教与能力深度都不在内）：

| 轴 | 管什么 | 角色 |
|---|---|---|
| **权力来源** succession | 谁来统治、怎么继承（世袭/选举/共和任期/神权/部落） | **主原型轴**——基本决定「叫什么政体」 |
| **中央-地方** centralization（=王权） | 中央能从产出里汲取多少、地方多自治（0–100 连续） | **同型变体轴**——压力驱动，动态漂移 |
| **财政制度** fiscal | 怎么收钱（领主代征/包税/直接征税/商业关税/游牧无税） | 决定金钱产出的方向与效率 |
| **军事制度** military | 怎么征兵（封建征召/常备军/全民皆兵/雇佣立国） | 决定动员成本与军需产出 |
| **立法机构** assembly | 有没有议会、议会多强（无/等级会议/议会主权） | 决定王权上限、改革速度、合法性 |

**少数轴定名、多数轴定变体**：`succession + assembly` 基本决定原型（君主/共和/神权/部落），其余轴在同名下自由变化。于是——
- 同是世袭君主：法（中央高、议会弱）→「绝对君主国」；英（中央中、议会主权）→「议会君主国」。**同型有别。**
- 异型之间天然不同：部落（部落推举+全民皆兵+游牧无税）vs 商业共和（共和任期+商业关税）。**异型有差。**

## C. 四条核心设计决策（讨论结论）

### C1. 王权：压力驱动 + 阶层调制方向

王权不再是滑块，而是**每季被压力推着漂移**，玩家用制度抉择去顶：

- **内部压力**（不满/低合法性/叛乱）→ 几乎总是推向**集权**（镇压冲动），代价是满意进一步崩。
- **外部压力**（战争/邻国威胁）→ 方向**由主导阶层决定**（Tilly「强制 vs 资本」）：
  - 贵族/平民/战士等**强制型**阶层主导 → 外压逼**集权**（榨取资源养兵，法国绝对主义路径）。
  - 商人/市民/商社等**资本型**阶层主导 → 外压逼**让权/宪政**（拿特权换军费，英国议会坐大路径）。

> 这条把一个惰性变量接回了核心循环 `压力层 → 反作用`，且用的全是现成变量（军事压力、邻国威胁、不满、阶层权力）。

### C2. 模块化的优劣势：每个选项是机器的不同形状，不是 +3% 旋钮

模块选项不止缩放数值，能**改方向、改流向、换机制**——这才是模块化的价值，也让五个原型彼此玩感不同。典型：

> **游牧**：`军事=全民皆兵` 把征召人口成本压到近零（动员极强），但 `财政=游牧无税` 同时把定居产出基底打掉（money×0.2 / food×0.5），靠劫掠与贡赋活命。一个强 buff，一个结构性 redirect。

### C3. 改革：压力挣来的「制度抉择」，不是随手点的菜单

改革 = 改某一个模块的取值，但**不能想点就点**：只有当 `外部压力 vs 内部压力` 越过窗口、**且阶层结构允许、且到了对应纪元**，才会弹出一条「制度抉择」二选一（复用既有裁断机制）。于是改革从「刷数值」变成「历史关口的抉择」——你在压力逼到头时选择集权应战还是让权筹饷，而不是攒够一个计数器就升级。

### C4. 阶层 / 国名 / 宗教 / 能力深度的归属

- **阶层随模块派生**：阶层**集合**跟着模块走（选游牧给牧民、选商业给商社），**分布**靠每国一份初始种子（法国贵族重、英国市民/议会重）→ 同政体也能不同分布。改模块时阶层按同系回流、权力守恒。
- **国名随政体动态变化**：政体变 → 国名变（法兰西王国 → 共和国 → 帝国）。技术上 `name` 是全引擎主键不可动，靠派生的 `displayName` 实现。
- **宗教剥离**：宗教不是政体结构，独立成「信仰」子系统；教士阶层留在国家，信仰政策反过来喂教士满意（跨系统流）。
- **改革槽废除**：不要抽象的「能力深度」轴；「先进」= 采纳了更先进的**制度本身**，技术先进度交给**科技系统**单独承载。

## D. 它如何回到核心循环

> 基底 → 流 → **压力层** →（压力驱动王权漂移 / 触发制度抉择）→ **制度模块**（调产出/动员/汲取阀）→ 反作用回基底 → 循环。

国家系统由此从「静态卡片」变成「被世界压力驱动、又反过来塑造产出的调节器」，玩家始终是**在压力关口扰动它**，而非凭空驱动它。

# 第二部分 · 落地蓝图

## 0. 本次设计的三个定调

1. **宗教剥离**：宗教（宽容/正统/改革宗）+ 宗教改革不再属于国家系统，独立成「信仰」子系统（§10）。国家系统五轴纯结构。
2. **改革槽彻底废除**：删除 `reforms` 6 槽与 `advanceReform`。产出差异不再来自「深度+3%/级」，而是来自**选了哪个制度模块**；「先进」= 采纳了更先进的制度本身。能力深度由**科技系统**单独承载，国家系统只管**结构选择**。
3. **doc 36 = 国家系统的权威设计**：本文覆盖国家系统的全部调整（数据结构、王权、改革、阶层、国名、产出机器、继承、UI 重构、消费方迁移、废止清单）。

## 0.1 设计原则

- **接着长不推倒**：现有 `laws`(税收/动员) + `assembly` 升格为模块；`authority` 法律溶解为派生（§13）。
- **每个模块必须挂流**：空转选项不准进（总纲三问）。
- **身份键不可变**：`country.name` 是全引擎主键（约 150 处），绝不重命名；动态名靠 `displayName` 派生（§7）。
- **改产出机器要有 sim 兜底**：§8 改造排在 `tools/sim`（34 号 P3）之后。
- **AI/测试同步迁移**：每个读 `government.*` 的消费方同阶段改，杜绝骨架政府崩溃。

  阶层结构调制)    └────┬─────────┬─────────┬─────────┬─────────┬─────────┘
                       │         │         │         │         │
            派生 ┌─────▼─────────▼─────────▼─────────▼─────────▼────┐
                 │ archetype → displayName（动态国名）             │
                 │ estateKeys（阶层集）× 国家分布种子 → 阶层       │
                 │ 各模块 effect → tileOutput/maintenance/mobilize │

## 4. 王权（中央-地方轴）：压力驱动 + 阶层调制

替换 `centralPower`（手动滑块）为每季自动漂移；玩家用制度抉择（§5）跳变。新增 `driftCentralization`，挂在 `processEstates` 后：

```js
function driftCentralization(world, polity) {
  const c = world.countries[polity], inst = c.government.institutions;
  const external = clamp(c.pressures.military + neighborThreatIndex(world, polity)); // 外压
  const internal = clamp(c.unrest + lowSatisfactionCount(c)*8 + Math.max(0, 40 - c.legitimacy)); // 内压
  const coercion = estatePowerShare(c, ["nobles","peasants","warriors","clans","princes"]);
  const capital  = estatePowerShare(c, ["merchants","companies","citizens","oligarchs","guilds"]);
  let target = inst.centralization;
  target += internal * 0.04;                          // 内压 → 集权（镇压冲动）
  target += external * 0.04 * (coercion - capital);   // 外压：强制型→集权 / 资本型→让权
  const cap = HIFI_INSTITUTIONS.assembly.options[inst.assembly.type].powerCap;
  inst.centralization = clamp(Math.min(cap, lerp(inst.centralization, target, 0.15)), 0, 100);
}
```

- 高 centralization 仍走 `crownCost`（>70 每季掉满意）做负反馈（保留现逻辑，字段改名）。
- `estatePowerShare/neighborThreatIndex/lowSatisfactionCount` 为新增纯函数。
- 现有读 `centralPower` 处全部改读 `institutions.centralization`（§12）。

## 5. 改革 = 压力门控的制度抉择（替代改革槽 + 决议）

**彻底取代** `advanceReform`（已删）与裸 `decisions`（并入此处）。新增 `processInstitutionForks(world)`，挂 `processHistory`，复用 35 号已落地的 `processPlayerCrises` 裁断机制：压力窗口 + 阶层条件 + 纪元条件满足才弹「制度抉择」，玩家二选一改一个模块。

```js
for (const fork of INSTITUTION_FORKS) {
  if (!fork.window(external, internal)) continue;   // 压力窗口
  if (!fork.estateGate(country)) continue;          // 阶层条件
  if (!fork.eraGate(world)) continue;               // 纪元条件
  if (!fork.fromState(country)) continue;           // 当前模块状态匹配
  pushInstitutionChoice(world, fork); break;        // 作为裁断弹出（单列冷却）
}
```

`INSTITUTION_FORKS`（**所有旧 decisions 与法律切换在此重生，gate 全部换成 纪元/阶层/压力，无改革槽**）：

| 抉择 | 从→到 | 压力窗口 | 阶层/纪元条件 | 选项 |
|---|---|---|---|---|
| 设立等级会议 | assembly none→estates_general | — | 可代表阶层权力≥阈值 · 纪元≥大发现 | 设立（合法性+）／维持 |
| 议会主权 | estates_general→parliamentary | external−internal ≥ 15 | 资本阶层主导 · 纪元≥革命前 | 让权（王权上限↓、改革快）／拒绝 |
| 绝对主义 | centralization 跳至 85 + assembly 弱化 | internal ≥ 20 | 强制阶层主导 · 纪元≥绝对主义 | 集权（贵族−）／维持 |
| 直接征税 | fiscal demesne/包税→direct | internal(财政) ≥ 30 | 中央≥50 | 改制（money×1.2、贵族−10）／维持 |
| 商业立国 | fiscal→commercial | — | 商社/港口阶层主导 | 改制（对外分成↑）／维持 |
| 常备军 | military→standing_army | external ≥ 35 | 货币化财政 · 纪元≥绝对主义 | 建军（军需↑维护↑）／维持 |
| 走向共和 | succession→republican_term | — | 议会主权 + 君主合法性持续低迷 · 纪元≥革命 | 共和（换原型/国名/阶层集）／维持 |

> 「走向共和」「绝对主义」这类**换原型**的抉择会触发 §6 阶层集迁移 + §7 国名重派生。

## 6. 阶层派生（集合 × 分布 × 演化）

替换 `createEstates(type)` 为 `deriveEstates(institutions, seed)`：

```js
function deriveEstates(institutions, seed) {
  const keys = unique([ ...succ.estatesBase, ...fiscal.estatesAdd, ...military.estatesAdd, ...assemblyExtra ]);
  return Object.fromEntries(keys.map(k => {
    const [label, power, sat, priv] = data.estates[k];
    const s = seed?.[k] || {};
    return [k, { label, power: s.power ?? power, satisfaction: s.satisfaction ?? sat, privileges:[...priv] }];
  }));
}
```

- **分布种子**落 `countries.js`：`estateSeed:{ nobles:{power:58}, commons:{power:30} }`。法国贵族重、英国市民/议会重 → 同原型不同分布。
- **改制迁移** `reconcileEstates(old, next)`：存活阶层延续满意/权力；新增按种子初始化；移除阶层权力按同系（贵族系/资本系/信仰系/平民系）比例回流，**总权力守恒**。
- 新阶层（electors/financiers/condottieri/herders/bureaucrats…）补进 `estateResource` 惩罚映射、颜色表、`codex` 词条。

## 10. 宗教剥离（移出国家系统）

| 移出项 | 去向 |
|---|---|
| 宗教法律（宽容/正统/改革宗） | 新「信仰」子系统的**信仰政策**（本文不展开，另立设计） |

**残留耦合（保留在国家侧）**：教士/教会阶层（clergy/church/imperial_church）仍是政体阶层集的一部分；信仰子系统的政策**反馈**到这些阶层的满意度（跨系统流：信仰政策 → 教士阶层满意 → 阶层惩罚回灌产出）。即：**阶层留在国家，宗教政策杆移到信仰**。神权原型（succession=theocratic）仍是国家结构，与信仰政策正交。
