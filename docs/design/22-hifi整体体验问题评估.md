# 22 - hifi 整体体验问题评估

结论：**hifi-demo 的美术气质、地图基础和系统深度已经成立，但整体体验还停在“复杂沙盘”阶段，没有完全进入“玩家知道自己要做什么、为什么做、做完有什么后果”的策略游戏阶段。**

当前最核心的问题不是机制少，而是四件事：

| 问题 | 一句话说明 |
|---|---|
| 目标感弱 | 玩家开局不知道本国这一局要追求什么 |
| 反馈弱 | 玩家操作后只看到数字变动，不清楚因果 |
| 压力弱 | 连续结束季度也会变强，战争和危机没有足够推到玩家脸上 |
| HUD 过重 | 桌面还能接受，横屏手机上主界面被面板吞掉 |

## 三、问题总表

| 编号 | 问题 | 影响 | 建议方向 |
|---|---|---|---|
| 1 | 开局目标感弱 | 玩家不知道法兰西这一局该干什么 | 做国家长期目标 + 本季三件事 |
| 2 | 御前会议不可执行 | 建议像提示文案，不像策略选择 | 改成可点击行动草案 |
| 3 | 战争没有进入问题队列 | 玩家不知道自己已经处于战争中 | 战争、军令、和平机会必须进入右侧队列 |
| 4 | 结束季度过于顺滑 | 玩家可连续刷资源，缺少风险 | 季报、预警、局势推进要打断空转 |
| 5 | 操作反馈太安静 | 玩家不知道刚才操作改变了什么 | 每次操作显示成本、收益、下季影响 |
| 6 | 外交列表过长 | 玩家不知道先找谁外交 | 邻国/敌国/可缔约对象优先，支持搜索筛选 |
| 7 | 非己方地块显示操作按钮 | 玩家误以为能建设、征兵、整合别国地块 | 非己方地块改为查看/外交/宣战 |
| 8 | 已建建筑仍可点 | 重复点击后才提示已有建筑 | 建成后按钮改“已建成”并禁用 |
| 9 | 军令按钮过度图标化 | 玩家要猜每个按钮含义 | 军团面板按钮显示短文字 |
| 10 | 地图工具过大 | 右下占地图，手机尤其严重 | 地图模式改成当前模式 + 展开选择 |
| 11 | 地块详情默认过重 | 左下长期遮挡地图 | 默认压缩，点击展开完整详情 |
| 12 | 资源增长缺少解释 | +318/季、+179/季看起来像纯数字 | 增加资源构成和季度账本 |
| 13 | 国家选择缺国旗/选中态 | 选择器有信息但识别感弱 | 加国旗、当前选中高亮、自动滚动到当前国家 |
| 14 | 错误反馈不够显性 | 目标不可达会进控制台，玩家无明确反馈 | 所有规则错误转成玩家可见 toast/提示 |
| 15 | 个别文案混淆 | “征集军需”显示军事点 +30 | 明确区分军事点与军需 |

## 四、目标感问题

目标感弱是当前最影响可玩性的体验问题。

### 4.1 当前表现

| 现象 | 玩家感受 |
|---|---|
| 开局显示“待办已清” | 好像没什么必须做 |
| 法兰西已有战争，但右侧问题为空 | 不知道战争正在影响自己 |
| 国家详情有时代处境，但没有目标 | 知道背景，不知道任务 |
| 御前会议给三条建议，但不绑定行动 | 看完仍然要自己找按钮 |
| 连续结束季度资源上涨 | 不操作也变强，削弱策略紧迫感 |

### 4.2 根因

| 根因 | 说明 |
|---|---|
| 缺少国家目标层 | 法兰西没有“收复阿基坦 / 压制勃艮第 / 稳固王权”这类显性目标 |
| 缺少中期议程层 | 玩家没有 8-12 季的阶段目标 |
| 缺少本季行动层 | 本季该做什么没有被翻译成 2-3 个按钮 |
| 问题队列规则太窄 | 只有阻断事件明显进队列，战争/外交机会没有形成待办 |
| AI 建议没有接行动目录 | 顾问建议没有直接对应合法行动 |

### 4.3 修复原则

| 原则 | 做法 |
|---|---|
| 目标由规则生成 | 不让 AI 凭空编任务 |
| AI 负责解释和包装 | AI 把规则目标说成幕僚建议 |
| 每条建议必须可执行 | 绑定跳转、执行或查看 |
| 长中短目标同时存在 | 国家使命、阶段议程、本季三件事 |
| 目标必须接地图 | 目标落到地块、军团、外交对象、阶层 |

## 五、AI 接入如何解决目标感

AI 可以解决“为什么要做”和“下一步怎么做”，但不能单独解决“游戏目标是什么”。正确方式是：**规则目标 + AI 顾问表达 + 行动草案校验**。

| 层级 | 规则负责 | AI 负责 | 玩家看到 |
|---|---|---|---|
| 长期目标 | 根据国家、时代、历史处境生成使命 | 解释这个目标为何重要 | “收复阿基坦：恢复王权威信” |
| 中期议程 | 根据当前资源、战争、阶层生成阶段任务 | 总结取舍和风险 | “整顿诺曼底：提升控制力与财政” |
| 本季建议 | 从行动目录选合法动作 | 用顾问口吻说明原因 | “财政官建议在鲁昂建市场” |
| 季度反馈 | 记录本季 state 变化 | 写成季报 | “本季国库增长主要来自诺曼底港口” |

### 5.1 第一版御前会议目标系统

| 区块 | 当前 | 建议 |
|---|---|---|
| 国家预警 | “没有迫近危机” | 加“仍在百年战争中 / 英格兰威胁 / 北境外交风险” |
| 顾问建议 | 三条文字 | 三张行动草案卡 |
| 世界局势 | 暂无大型局势 | 显示正在进行战争、相邻强国动向、贸易机会 |
| 操作按钮 | “垂帘听政 4 季” | 改为“执行建议 / 查看目标 / 跳转面板” |

### 5.2 行动草案示例

| 顾问 | 建议 | 成本 | 预期收益 | 操作 |
|---|---|---|---|---|
| 财政官 | 在鲁昂修建市场 | 24 金钱 + 1 行政点 | 金钱产出提升，市场能力增强 | 跳转经济建设 |
| 外交官 | 派使节稳住勃艮第 | 1 外交点 + 1 使节 | 降低北境被夹击风险 | 直接派使节 |
| 军务官 | 主力军推进第戎或防守巴黎 | 1 军令 | 明确百年战争路线 | 打开军团面板 |

### 5.3 必须坚持的边界

| 边界 | 说明 |
|---|---|
| AI 不直接改 state | 只能输出 proposal |
| proposal 必须进规则校验 | 资源、地块归属、可达性、外交状态都要校验 |
| 校验失败直接暴露 | 不做“智能兜底” |
| 建议来源可追溯 | 每条建议显示依据：资源、地块、战争、关系、阶层 |

## 六、界面问题

### 6.1 HUD 占用过高

| 场景 | 观察 | 影响 |
|---|---|---|
| 桌面 1280×720 | HUD 粗略占屏约 36% | 还能接受，但地图主角感被削弱 |
| 横屏手机 932×430 | HUD 粗略占屏约 79% | 微信小游戏横屏基本被 UI 吃掉 |

建议：

| 区域 | 当前 | 改法 |
|---|---|---|
| 顶部资源栏 | 7 个资源全展开 | 只显示核心 4 个，次级资源点开展开 |
| 左下地块详情 | 默认完整展开 | 默认迷你卡，点击展开 |
| 右下地图工具 | 10 模式常驻 + 小地图 | 只显示当前模式，点开模式盘；小地图默认收起 |
| 右侧问题面板 | 空状态也占位 | 无问题时折叠为小按钮 |
| 底部命令坞 | 常驻 6 个外交命令 | 按选中对象变化；无对象时收起 |

### 6.2 地块面板操作不分归属

| 当前 | 问题 |
|---|---|
| 点击布列塔尼/勃艮第地块仍显示建设、征兵、整合、派驻 | 玩家会误以为可直接操作别国地块 |
| 进入经济建设后才显示“非己方地块” | 提示太晚 |

建议：

| 地块类型 | 默认动作 |
|---|---|
| 己方地块 | 建设、征兵、整合、派驻 |
| 外国和平地块 | 查看国家、外交、宣战、标记目标 |
| 外国交战地块 | 进军、围城、查看战争目标 |
| 海域地块 | 舰队/贸易/探索，当前无海军时只读 |

### 6.3 地图模式过密

| 当前模式 | 问题 |
|---|---|
| 政治、地形、人口、产出、贸易、宗教、家族、政体、阶层、军事全部常驻 | 模式完整，但视觉负担大 |

建议改为：

| 一级 | 二级 |
|---|---|
| 政治 | 国家、政体、家族、阶层 |
| 地理 | 地形、气候、控制力 |
| 经济 | POP、产出、贸易品、市场 |
| 军事 | 军团、占领、补给、战争目标 |

## 七、游玩体验问题

### 7.1 反馈链不完整

| 动作 | 当前反馈 | 缺少 |
|---|---|---|
| 强化征税 | 行政点、金钱、合法性变化 | “本次 +25 金钱、-2 合法性”的确认 |
| 建农庄 | 资源扣除 | “鲁昂已建农庄，下季粮食 +X”的提示 |
| 派使节 | 外交点和使节数变化 | “使节已派往阿拉贡，预计每季信任 +X” |
| 规划军团路线 | 地图上有路线迹象 | “军令已下达：鲁昂 → 第戎，预计 X 季” |
| 结束季度 | 资源上涨 | 季报：钱、粮、军需分别来自哪里 |

### 7.2 压力系统没有推到玩家面前

| 现象 | 问题 |
|---|---|
| 百年战争进行中，但右侧问题队列为空 | 战争没有形成待办 |
| 资源高速上涨 | 没有战争消耗、阶层压力、外交机会打断 |
| 御前会议显示无危机 | 与“进行中战争 1”冲突 |

建议：

| 队列类型 | 条件 |
|---|---|
| 战争待办 | 有进行中战争、敌军接近、战争目标可推进、可议和 |
| 外交机会 | 可结盟、可贸易、邻国威胁上升 |
| 内政压力 | 控制度低、阶层满意下滑、合法性下降 |
| 经济机会 | 有高收益建设点、资本池可投资、贸易路线可升级 |

### 7.3 数值膨胀削弱紧迫感

测试中连续推进季度，粮食、金钱、军需快速增长。问题不一定是数值错，而是**增长没有目标消耗**。

| 资源 | 当前体验 | 需要的出口 |
|---|---|---|
| 粮食 | 很快堆高 | 军队维护、灾害、城市增长、救济 |
| 金钱 | 很快堆高 | 建设、外交、佣兵、行政维护 |
| 军需 | 很快堆高 | 常备军维护、训练、补给、炮兵 |
| 行动点 | 很快到上限 | 更多高价值决策和季度计划 |

## 八、具体缺陷清单

| 缺陷 | 影响 | 建议 |
|---|---|---|
| `征集军需` 文案显示“军事点 +30” | 混淆军事行动点与军需 | 改成“军需 +30” |
| 建筑建成后仍显示可建设按钮 | 重复点击才知道不可建 | 建成后禁用并显示已建成 |
| 不可达军令进控制台错误 | 玩家不知道失败原因 | 显示“目标不可达”，退出规划态 |

| 外交对象默认阿拉贡 | 不一定符合当前局势 | 默认优先敌国、邻国、战争相关国家 |
| 国家选择器缺国旗 | 身份识别弱 | 候选卡加旗帜/盾徽 |
| 当前国家选择不明显 | 预览是法兰西，候选带停在阿拉贡开头 | 自动滚到当前国家并高亮 |

## 十、推荐下一版体验目标

下一版不追求新增大系统，只追求玩家进入游戏 30 秒内能回答三个问题：

| 问题 | 页面应该给出的答案 |
|---|---|
| 我是谁 | 法兰西王国，腓力六世，瓦卢瓦家族，正在封建纪元 |
| 我要干什么 | 收复阿基坦、稳住勃艮第、整顿诺曼底 |
| 本季先做什么 | 建设鲁昂、派使节到勃艮第、给主力军下达防守/推进军令 |

如果这三件事成立，AI 接入才有价值。否则 AI 只会变成“会说话的提示框”。
