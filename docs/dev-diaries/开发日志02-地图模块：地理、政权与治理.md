# 开发日志 02 - 地图模块：地理、政权与治理

大家好，今天聊地图。

在《帝国的代价》里，地图不是背景图，也不是单纯用来摆军队的棋盘。地图本身就是游戏规则的一部分。

我们的目标很直接：

**让每个地块都同时表达地理、人口、政权、经济和治理成本。**

## 时代背景

第一版地图采用 **1337 年前后** 作为开局背景。

这是一个很适合策略游戏的时间点。中世纪晚期的旧秩序还没有消失，近世国家的集中化、财政化和全球化已经开始露出苗头。

| 时代条件 | 地图上的含义 |
|---|---|
| 百年战争即将展开 | 法兰西、英格兰和低地不是平静边界，而是长期战争空间 |
| 黑死病即将冲击欧洲 | 高人口地区既是财富核心，也是灾害放大器 |
| 拜占庭仍在但衰弱 | 君士坦丁堡、海峡和巴尔干必须成为战略焦点 |
| 奥斯曼正在崛起 | 安纳托利亚和巴尔干不能只是边缘地带 |
| 意大利城邦繁荣 | 城市、市场、港口比单纯土地面积更重要 |
| 北非连接地中海贸易 | 地中海不是留白，而是政治和经济通道 |

所以我们的第一张地图不是“欧洲陆地 + 一圈蓝色背景”。欧洲、北非、地中海、海峡、港口和海域都必须被玩家看见，也必须能被规则使用。

## 为什么用六边形

我们选择六边形，不是因为它看起来像桌游，而是因为它适合在手机屏幕上表达区域控制。

| 问题 | 六边形的解决方式 |
|---|---|
| 移动方向容易失衡 | 六个方向距离更接近，移动判断更自然 |
| 补给和控制力需要稳定相邻关系 | 每格邻接清楚，适合做道路、市场、补给和扩张 |
| 真实省份太碎，手机难点选 | 六边形能保留地理轮廓，同时让操作足够可靠 |
| 地中海需要参与玩法 | 海域也能被填成可移动、可封锁、可连接的地块 |

但六边形不能变成整齐的矩形棋盘。

当前原型以欧洲、北非和地中海的真实轮廓为基础，用六边形填充陆地与海域。直布罗陀海峡、英伦三岛、意大利半岛、巴尔干和小亚细亚这些形状，都需要在地图上保留可读性。

## 地块不是城市

地图模块最容易犯的错，是把每个地块都当成一座城市。

本作不会这样做。

**地块是可治理的地理单元，城市只是其中少数地块上的锚点。**

| 类型 | 含义 | 玩法作用 |
|---|---|---|
| 普通地块 | 农村、山地、森林、边疆、海岸等空间 | 提供产出、移动、控制和风险 |
| 城市地块 | 有明确历史城市的地块 | 市场、建筑、人口密度和地区价值锚点 |
| 首都地块 | 政权中心 | 用星星标记，影响合法性、控制力和战略目标 |
| 海域地块 | 可航行海上空间 | 舰队移动、封锁、登陆、港口连接 |

巴黎、伦敦、佛罗伦萨、君士坦丁堡这样的地点会作为城市出现。普通地块仍然有地区名、地形、气候、POP、归属和产出，但它不等于一座城市。

这样做的目的很简单：让地图既有历史锚点，也保留大尺度治理空间。

## 四种地图模式

当前原型支持四种地图模式：政治、地形、人口、产出。

默认是 **政治模式**。玩家打开游戏时，第一眼应该知道“谁在统治哪里”。

| 地图模式 | 展示内容 | 玩家主要判断 |
|---|---|---|
| 政治模式 | 政权归属、首都、己方与敌方边界 | 战略格局、扩张方向、外交压力 |
| 地形模式 | 平原、森林、丘陵、山地、沙漠、草原、湿地、海岸、海域 | 移动成本、战争风险、基础产出 |
| 人口模式 | POP 总量和密度 | 税收、兵源、灾害风险、阶层压力 |
| 产出模式 | 贸易品和资源倾向 | 建设方向、市场价值、战争目标 |

这四种模式对应四种问题。

| 玩家问题 | 对应模式 |
|---|---|
| 这里属于谁？ | 政治模式 |
| 这里天然适合做什么？ | 地形模式 |
| 这里有多少社会力量？ | 人口模式 |
| 这里为什么值得争夺？ | 产出模式 |

地图模式不是换皮，而是让玩家从不同角度理解同一片土地。

## 地理底层

每个陆地地块都由一组底层条件构成。

| 条件 | 影响 |
|---|---|
| 地形 | 决定基础粮食、金钱、军需和移动成本 |
| 气候 | 改变粮食稳定性，寒冷和干旱地区更难供养人口 |
| 河流 | 提升农业、交通、市场接入和控制力 |
| 海岸 | 提供港口、贸易、海军和登陆价值 |
| 邻近度 | 影响控制力、市场连接和行政成本 |
| POP | 决定劳动、税源、兵源和阶层力量 |
| 建筑 | 把地理潜力转化为稳定产出 |
| 市场接入 | 决定产出能不能变成国家财富 |

我们想保留《欧陆风云 5》最重要的地图思想：地理不是几条修正值，而是国家能力的底座。

山地边疆可能适合防守，但税收差、移动慢、整合难。  
海岸城市可能很富，但会受舰队、封锁和港口控制影响。  
高人口农田能提供粮食和兵源，但黑死病、征兵和饥荒也会更危险。

## POP 与地块

POP 是地图活起来的关键。

当前原型把 POP 简化成四类。

| POP 类型 | 主要价值 | 主要风险 |
|---|---|---|
| 农民 | 粮食、基础劳力、征兵来源 | 被征兵和劳役压垮后容易动乱 |
| 市民 | 金钱、市场、城市发展 | 会推动商人阶层权力上升 |
| 贵族 | 军事、骑兵、地方影响力 | 会阻碍中央集权和改革 |
| 教士 | 合法性、安抚、文化整合 | 会限制改革和异端宽容 |

这套 POP 不是为了让玩家做人口表，而是为了让同一块土地在不同国家手里产生不同结果。

有道路、有市场、有控制力的国家，可以把人口变成税收、粮食和军队。控制力不足的国家，只会看见腐败、叛乱和低效。

## 控制力与邻近度

扩张游戏最容易犯的错，是让玩家只关心“拿到了没有”。

本作不把拿地当成结束。拿到土地只是开始。

| 状态 | 含义 | 收益 | 风险 |
|---|---|---|---|
| 占领 | 军队暂时站住 | 阻止敌人使用 | 几乎不稳定产出 |
| 控制 | 国家开始征税、征兵和建造 | 获得部分收益 | 叛乱和行政压力上升 |
| 整合 | 地区进入国家机器 | 稳定产出 | 消耗时间、官员和政治资源 |

控制力来自首都距离、道路、港口、地形阻隔、海洋连接和行政投入。

这会制造一个重要选择：

**远方的富地不一定是好地。**

如果玩家拿下一个高价值海港，却没有舰队、道路和行政官维持连接，那么这个海港可能不是财富，而是财政黑洞和叛乱温床。

## 海洋也是地图

地中海在本作中不是蓝色背景，而是一组可操作地块。

| 海域作用 | 对玩法的影响 |
|---|---|
| 舰队移动 | 让威尼斯、热那亚、阿拉贡、拜占庭、马穆鲁克有不同路线 |
| 港口连接 | 让控制力和市场可以跨海延伸 |
| 海峡阻断 | 直布罗陀、英吉利海峡、博斯普鲁斯不会被陆地错误连通 |
| 海上封锁 | 富裕海岸城市会因为制海权变化而失去收益 |
| 登陆威胁 | 地中海强国可以绕开陆路防线制造压力 |

这也是为什么第一版必须包含欧洲、北非和地中海。没有海，很多国家的战略就会失真。

## 地块详情

点击地块后，左下角会打开地块详情抽屉。

我们没有把它做成一张冷冰冰的表格，而是在顶部放一张地形横幅。平原、森林、丘陵、山地、沙漠、草原、湿地、海岸和海域都有自己的时代油画风格图片。

| 信息 | 展示方式 |
|---|---|
| 地块名称和所属地区 | 横幅标题和副标题 |
| 气候 | 图标和文字 |
| 所属政权 | 同时代风格旗帜标记 |
| POP | 总数和四类人口缩写 |
| 首都 | 地图上用星星标记 |
| 地块细节 | 横幅下展示地形、建筑、控制力和产出推导 |

这样做是为了让玩家先知道“这是什么地方”，再理解“它怎么算”。

## 它如何影响其他系统

地图模块不是独立功能。它会牵动几乎所有核心系统。

| 地图信息 | 影响系统 | 结果 |
|---|---|---|
| 地形和气候 | 经济、战争 | 决定产出、移动成本和作战风险 |
| POP | 经济、阶层、军事 | 决定税源、兵源和社会压力 |
| 海岸和海域 | 贸易、海军、外交 | 决定港口价值、封锁和登陆威胁 |
| 控制力 | 财政、叛乱、整合 | 决定土地能否真正变成国家能力 |
| 市场接入 | 经济、建设 | 决定资源能不能流入国家网络 |
| 首都和城市 | 政治、战略目标 | 决定合法性中心和战争价值 |

地图越真实，玩家的决策就越不应该只有“数值最大化”一种答案。

当前版本仍是原型，不是最终历史数据库。后续重点是继续校准地块归属、城市位置、人口密度、贸易品分布和海峡通行规则。

## 本篇结论

地图模块的设计原则是：

**地图先表达历史，再承载规则，最后服务操作。**

如果地图只好看，它只是壁纸。  
如果地图只有数值，它会变成表格。  
我们要做的是第三种：让地理、人口、政权和经济在同一张地图上互相解释。

下一篇开发日志会讲政权与阶层：为什么国家内部的贵族、商人、教会和农民，会像边境敌人一样改变玩家的战略。
