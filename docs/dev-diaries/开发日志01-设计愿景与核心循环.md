# 开发日志 01 - 设计愿景与核心循环

大家好，今天聊《帝国的代价》的设计愿景。

在这款游戏里，国家不是一张收入表，也不是一组可以随意加点的数值。国家是一台会被人口、阶层、地理、战争和危机不断拉扯的机器。

我们的目标很直接：

**做一款桌游式中重策略微信小游戏，让玩家体验“扩张很诱人，但治理更困难”。**

## 设计背景

我们选择的不是轻量卡牌手游方向，而是“桌游玩法 + 微信小游戏载体”。

桌游擅长表达取舍。玩家能清楚看见地图、单位、资源、阶层轨道和危机压力。微信小游戏擅长处理结算。它可以自动计算补给、市场、控制力、人口变化和叛乱风险。

| 载体能力 | 在本作中的作用 |
|---|---|
| 桌游式地图 | 让扩张、围城、封锁、移动和区域控制变得可见 |
| 微信小游戏结算 | 自动处理财政、粮食、军需、POP、控制力和事件 |
| 短局结构 | 单局控制在 30-60 分钟，支持暂停和续局 |
| 触屏交互 | 地图缩放、地块详情、抽屉面板和行动确认更轻 |

所以本作不是“把桌游搬进手机”，也不是“做一款卡牌化大战略”。它更像一张会自动结算的历史策略桌。

## 玩家扮演什么

玩家扮演的不是某个国王、家族或英雄。

玩家控制的是一个国家背后的长期统治集团。君主会更替，阶层会反抗，法律会改革，战争会改变边界，但玩家代表的是这个国家持续数代的战略意志。

这件事很重要，因为它决定了游戏的视角。

| 视角 | 本作不采用的原因 |
|---|---|
| 单个君主 | 太容易变成角色扮演，国家系统会退到背景 |
| 一个家族 | 会把继承和婚姻变成核心，不适合本作第一版 |
| 一个抽象国家 | 太冷，玩家感受不到社会压力 |
| 长期统治集团 | 能同时解释改革、阶层妥协、战争动员和国家延续 |

你不是在问“我这个国王想做什么”，而是在问：

**这个国家机器还能承受多少扩张、改革和战争？**

## 为什么做这个系统

许多大战略游戏会让玩家习惯一个简单循环：打地、赚钱、再打更多地。

我们想让这个循环变得更有代价。

| 常见扩张循环 | 本作希望加入的压力 |
|---|---|
| 打赢战争 | 战争消耗人口、粮食、军需和合法性 |
| 获得土地 | 新地区需要控制、道路、官员和整合 |
| 获得收入 | 收入取决于市场接入、POP、建筑和稳定 |
| 继续扩张 | 阶层会索要回报，边疆会积累叛乱 |

扩张仍然应该令人兴奋，但它不能是无条件正确答案。

一块富裕土地如果离首都太远、道路不足、文化宗教复杂、控制力很低，它可能不是奖励，而是新危机的起点。

## 核心循环

本作的核心循环从人口和地理开始，而不是从金币开始。

```mermaid
flowchart LR
    A["地理与POP"] --> B["生产"]
    B --> C["财政 / 粮食 / 军需"]
    C --> D["建设 / 改革 / 征兵"]
    D --> E["国家能力"]
    E --> F["战争 / 外交 / 扩张"]
    F --> G["新地区"]
    G --> H["控制力与整合压力"]
    H --> I["阶层不满 / 叛乱 / 财政负担"]
    I --> D
```

这条循环里最重要的不是“变强”，而是“变复杂”。

| 循环阶段 | 玩家获得什么 | 玩家承担什么 |
|---|---|---|
| 生产 | 粮食、金钱、军需 | 依赖地形、气候、市场和人口 |
| 建设 | 更高产出、更好道路 | 建筑成本、阶层利益变化 |
| 改革 | 更强国家能力 | 贵族、教会、商人或农民反弹 |
| 征兵 | 军队扩张 | POP 损耗、农村压力、财政负担 |
| 扩张 | 新地块、新港口、新资源 | 控制力、整合、叛乱和补给问题 |

扩张不是循环的终点，而是循环变难的方式。

## 系统如何运作

第一版会把复杂历史模拟压缩成几组玩家能读懂的桌游式系统。

| 系统 | 作用 | 玩家主要取舍 |
|---|---|---|
| 地图 | 展示地形、政权、人口、产出、海域和首都 | 去哪里扩张，哪里值得守，哪里太难管 |
| POP | 提供粮食、税收、兵源和阶层力量 | 是发展人口，还是透支人口 |
| 阶层 | 代表贵族、商人、教会、农民的利益 | 妥协换稳定，还是改革换效率 |
| 建筑 | 把地理潜力变成稳定产出 | 建市场、道路、要塞还是农庄 |
| 控制力 | 决定国家能不能有效统治地块 | 快速占领，还是慢慢整合 |
| 战争 | 改变边界和力量平衡 | 兵力、补给、围城、财政能否撑住 |
| 危机 | 打断完美计划 | 如何在黑死病、继承、叛乱和战争中保住国家 |

这些系统不会各玩各的。

修路会影响市场，也会影响军队移动和控制力。征兵会增强军队，也会减少 POP、推高农民不满。建市场会增加金钱，也会增强商人阶层。改革能提高国家能力，也会触怒旧特权集团。

## 玩家会做什么选择

本作希望让玩家经常面对没有完美答案的选择。

| 玩家选择 | 直接收益 | 长期代价 |
|---|---|---|
| 继续打仗 | 获得土地、打击敌人 | 军需下降、POP 损耗、厌战和财政压力 |
| 停战治理 | 恢复控制力、整合新地 | 给敌人喘息时间，短期扩张变慢 |
| 增税 | 获得现金 | 商人或农民不满，繁荣下降 |
| 建道路 | 提升控制力和市场接入 | 消耗资金，可能让敌军也更容易推进 |
| 扶持贵族 | 获得将领和骑兵 | 中央集权变难 |
| 推动改革 | 国家能力上升 | 阶层反弹，合法性承压 |

最好的玩家不是扩张最快的人，而是能判断国家什么时候该前进、什么时候该停下来修复内部的人。

## 它如何影响其他系统

设计愿景会贯穿后续所有模块。

| 模块 | 受核心循环影响的方式 |
|---|---|
| 地图 | 每个地块都是治理对象，不只是收益格子 |
| 经济 | 财富来自 POP、建筑、市场和控制力的组合 |
| 战争 | 军队来自人口，胜利会消耗国家内部 |
| 外交 | 外部关系是战争和扩张成本的一部分 |
| 阶层 | 国家每一次强化都会改变社会集团的力量 |
| 危机 | 黑死病、叛乱、继承危机会惩罚失衡国家 |
| UI | 微信小游戏负责把复杂因果讲清楚，而不是隐藏起来 |

本作的复杂度不应该来自“玩家要记住很多小数值”，而应该来自“每个决定都会推动几个系统同时变化”。

## 当前原型

第一阶段原型已经围绕地图和核心循环展开。

| 内容 | 状态 |
|---|---|
| 欧洲 + 北非 + 地中海六边形地图 | 已完成可交互原型 |
| 政治 / 地形 / 人口 / 产出模式 | 已完成原型 |
| POP、建筑、控制力、市场接入 | 已进入地块产出推导 |
| 左上政权操作抽屉 | 已完成原型 |
| 右上国家资源 HUD | 已完成原型 |
| 左下地块详情抽屉 | 已完成原型 |
| 危机、阶层、战争完整循环 | 待下一阶段细化 |

第一版原型最重要的验证不是内容量，而是玩家会不会在“继续扩张”和“停下来治理”之间犹豫。

如果这个犹豫成立，游戏就有了核心。

## 本篇结论

《帝国的代价》的设计原则是：

**用地图点燃野心，用人口和阶层制造代价，用危机打断玩家的完美计划。**
